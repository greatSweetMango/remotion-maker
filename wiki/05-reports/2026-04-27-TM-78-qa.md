# TM-78 — Customize Undo/Redo QA

## TL;DR

**검증 결과: BLOCK (기능 미구현).**
Customize 패널의 슬라이더/색상 변경에 대한 undo/redo는 **현재 코드에 존재하지 않는다**. 키보드 단축키(Cmd+Z / Cmd+Shift+Z) 핸들러도 없고, `state.paramValues` 변경은 단순 덮어쓰기만 한다. TM-44의 PARAMS 라운드트립 위에 history layer가 추가되어야 한다.

## Verification scope (spec)

1. 50회 변경 → 50회 undo → 모두 초기 상태 복원
2. 30회 변경 → 10회 undo → 새 변경 → redo 시도 → 차단(분기)
3. history depth limit (예: 100) 도달 시 동작
4. 메모리 누수 (각 변경마다 history grow)

위 4개 항목은 **현재 구현에서 모두 N/A** — undo/redo 기능 자체가 없으므로 검증 대상 없음.

## Source-of-truth evidence

### 1. `studioReducer.UPDATE_PARAM` — history 트래킹 없음
`src/hooks/useStudio.ts:47-51`

```ts
case 'UPDATE_PARAM':
  return {
    ...state,
    paramValues: { ...state.paramValues, [action.payload.key]: action.payload.value },
  };
```

`paramValues`는 단순 덮어쓰기. 이전 값을 보관할 stack/ring buffer가 없음.

### 2. `versions[]` ≠ paramValues history
`src/hooks/useStudio.ts:52-81` (`ADD_VERSION`)

`versions[]`는 LLM **edit** 결과(코드 + parameters 정의)에 대한 snapshot이다. **PARAMS 값 변경**은 `versions[]`에 추가되지 않는다. 즉 HistoryGraph가 보여주는 history와 customize undo는 **별개의 차원**이며, 후자는 부재.

### 3. Cmd+Z / Cmd+Shift+Z 핸들러 부재
`grep -rn "metaKey\|ctrlKey\|KeyZ" src/`:

- `Studio.tsx:263` — `SequenceHotkeys`는 `if (e.metaKey || e.ctrlKey || e.altKey) return;` 즉 modifier가 있으면 즉시 무시.
- `PromptPanel.tsx:189` — Cmd+Enter는 generate에 사용. Cmd+Z 미처리.
- 그 외 어디에도 `'z'` 키 핸들러 없음.

### 4. UI에 undo/redo 버튼 없음
`CustomizePanel.tsx`, `Studio.tsx` header, `HistoryGraph.tsx` 모두 PARAMS 변경에 대한 undo/redo UI 컨트롤을 노출하지 않는다. HistoryGraph는 LLM edit version graph만 표시.

## Recommendation

- **REQUEST_CHANGES** — TM-44 후속으로 customize history layer 설계 및 구현 필요.
- 권장 아키텍처:
  - `useStudio` reducer에 `paramHistory: { past: PV[], future: PV[] }` 추가, `UPDATE_PARAM` 시 `past.push(prev); future = []` (분기 동작), depth limit (e.g. 100) 도달 시 oldest shift.
  - 새 액션: `UNDO_PARAM`, `REDO_PARAM`.
  - `Studio.tsx`에 window keydown handler — `cmd+z` → UNDO_PARAM, `cmd+shift+z` → REDO_PARAM. INPUT/TEXTAREA focus 시 native 동작 위임.
  - `CustomizePanel` 헤더에 ↶ ↷ 아이콘 버튼 (disabled 상태 = past/future 비어있음).
  - 같은 key 연속 변경(슬라이더 드래그)에 대한 debounce/coalesce — drag end 또는 250ms idle에 1번 history push.
