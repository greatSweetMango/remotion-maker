# TM-46 r5 회고

## 잘 된 점

1. **N=2 measurement 도입** — r1~r4 동안 단발 회차로 acceptance gating 했던 구조의
   한계를 r5 에서 처음으로 양적 측정. mean std=8.10, max std=34.50 — gate band(75±5)
   대비 측정 노이즈가 큰 상태였음을 데이터로 확인.
2. **TM-70 부분 효과 분리 검증** — judge 만 deterministic 화 했음에도 end-to-end
   variance 가 잔존. capture-side LLM code-gen variance 가 dominant 임을 확인.
   이전엔 judge variance 와 합쳐 보였는데 r5 가 둘을 분해해 보여줌.
3. **r4 회귀 가설 기각** — r3 71.2 → r4 63.4 가 실제 회귀가 아니라 단발 측정 노이즈였음을
   r5 N=2 평균 67.8 + std 8.10 으로 통계적 기각. TM-60~63 fix 머지가 점수에 negative
   효과 줬다는 잘못된 결론을 방지.
4. **자동화 + 비용 cap 내** — 30 prompts × 2 captures + 60 judge multimodal = $3.6
   (예상 $2 대비 +80% 지만 N=2 추가분이라 ROI 정당화). 사람 개입 0.
5. **measurement methodology insight** — capture variance 가 prompt 복잡도와 양의 상관.
   simple prompts (spinner, dots, ring, step indicator) 는 std=0~1.5, complex
   (bar race, 3-icon infographic, time-progress loader) 는 std=20~35.

## 아쉬운 점

1. **acceptance MISS 또 (67.8 < 75)** — 4회차 연속. 단 이번에 측정도구 한계가 명확해짐.
2. **capture-side temperature pinning 미적용** — gpt-4o-mini (code-gen) 호출에
   temp=0 + seed 를 줘서 capture variance 도 감소시킬 수 있을 가능성. 본 r5 에선
   미시도 (TM-70 은 judge 만 다룸).
3. **acceptance gate 재정의 미수행** — gate 가 측정도구 노이즈보다 작은 band 임이
   확인됐지만 본 PR 에서 gate 정의를 바꾸지는 않았음 (별도 ADR 필요).
4. **수동 collapse 케이스 PNG 검사 미수행** — dv-08 (std=34.5), ig-02 (27.5), ig-04 (31)
   각 PNG 셋을 시각 비교하면 "왜 다른가" 더 잘 이해할 수 있었을 텐데 시간 부족.
5. **N=2 는 std 추정에 약함** — 통계적으로 std 추정에는 N≥3 필요. 본 측정은
   "variance 가 크다는 정성적 사실 확인" 까지만 신뢰 가능.

## 다음에 시도

1. **capture-side determinism (`TM-72?`)** — generate 호출에 temp=0 + seed 넣고
   재측정. capture variance 가 0 에 수렴하는지 확인.
2. **acceptance gate v2 ADR** — 단발 avg ≥ 75 → N=3 평균 ≥ 75 AND mean_std ≤ 5.
   기존 acceptance pool 에서 per-prompt mean_std > 15 인 케이스 (dv-02, dv-03,
   dv-08, ig-02, ig-04, ld-03, ld-09) 는 분리 평가 (reference 템플릿 필요).
3. **야간 QA 1차 사이클 종결** — TM-46 acceptance MISS 지만 측정도구 한계가 명확해진
   시점에서 추가 회차 ROI 낮음. 본 PR 머지 후 다음 사이클로 위임.
4. **capture-PNG 자동 diff** — 같은 prompt 두 회차의 PNG 를 자동 비교 (pixel diff +
   embedding sim) 해서 capture variance 의 정량 재현.
