# TM-70 — TM-46 r3→r4 회귀 root cause analysis

## TL;DR

- **TM-46 r3 (avg 71.2) → r4 (avg 63.4) -7.8 회귀의 dominant 원인은 judge non-determinism**.
- gpt-4o judge 가 `temperature` 미설정(= OpenAI default **1.0**) + seed 미사용으로 호출됨.
  identical PNG triple 에 대해 **3회 반복 시 평균 ±10.4 점, 최대 ±17 점** drift 관측.
- **r3→r4 단일 회차 차이(-7.8)는 단일 샘플 noise 폭(10.4) 보다 작음** → 실제 회귀라고 단정 불가.
- 권고: judge 호출에 `temperature: 0, seed: 42` 강제 (본 PR 동봉) + 향후 acceptance gate 는
  N-shot 평균(N≥3) 으로 변경. ADR-PENDING-TM-70 으로 정책 박제.

## 가설 검증

| 가설 | 결과 | 근거 |
|---|---|---|
| (1) judge variance | **STRONG** | aggregate std 9.25 (default) → 5.81 (temp=0). per-sample max-min 14/17/10/6/5. |
| (2) prompt set/category mismatch | partial | r3→r4 일부 카테고리(transition/infographic) 가 TM-60~63 fix 대상이 아니었음. 단 "회귀" 가 아니라 "기대했던 개선이 없음" 으로 재해석. |
| (3) 실제 회귀 (TM-67/68/69 머지가 깨뜨림) | **WEAK** | 측정 노이즈(±10) 안에 포함. data-viz 는 +11 개선(TM-69 효과 일치), transition/infographic 가 같은 폭으로 노이즈 가능. 코드 회귀 증거 없음. |

## 실험 설계

### 입력
- 5 fixed sample × 3 frame PNG triple (TM-43 박제 스크린샷 재사용 — 캡처 재현 비용 0).
- TM-46 카테고리 매핑:
  - `tr-10-proxy` ← `zoom-transition` (transition)
  - `ig-01-proxy` ← `timeline` (infographic)
  - `dv-06-proxy` ← `progress-circle` (data-viz)
  - `ta-02-proxy` ← `typewriter` (text-anim)
  - `ld-01-proxy` ← `circle-pulse` (loader)
- 각 sample 의 PNG bytes 는 호출마다 동일 → 변동량은 100% judge-side.

### 절차
- **Condition A (현재 r3/r4 설정 재현)**: `temperature` 미설정 (default 1.0), no seed.
- **Condition B (제안 fix)**: `temperature: 0`, `seed: 42`.
- 각 sample × 각 condition × 3회 반복 = 30 OpenAI 호출.
- judge: gpt-4o, multimodal, 동일 system prompt + 동일 4축 rubric (`tm-46-judge.ts`).
- 도구: `__tests__/benchmarks/tm-70-judge-variance.ts` (본 PR 신규).

aggregate Δmax 10.4 = "동일 입력으로 3회만 호출해도 단일 샘플이 평균 10.4 점 뒤흔들림".

### 함의

- **r3 → r4 회차별 차이의 95% CI 가 [-15, +15]** 수준이라고 봐야 함 (n=29, sample-wise std≈10 가정).
  관측된 -7.8 은 noise 범위 안.
- **transition -13.5 / infographic -15.7 도 단일 샘플 Δmax 14/17 과 같은 자릿수** → 단일 회차로 카테고리
  성능 비교 자체가 통계적으로 무리.
- 즉 "TM-67/68/69 머지가 깨뜨렸다" 는 결론을 데이터가 지지하지 않음. 동시에 "TM-60~63 가 끌어올렸다"
  도 지지하지 못함. data-viz +11.0 만 TM-69 (json-mode) 의 generate 성공률 회복으로 설명 가능.

## 권고 (ordered)

1. **즉시 (본 PR 포함)**: `tm-46-judge.ts` 에 `temperature: 0, seed: 42` 추가.
   - 코드 변경: 5 lines.
   - 비용 무영향, 응답 속도 동일.
2. **다음 r5 회차**: 동일 30-prompt × 동일 PNG 셋을 새 judge config 으로 재채점 → r4 score 가
   noise 였는지 확정. PNG 재캡처 불필요 (TM-46 r4 결과 PNG 재사용).
3. **acceptance gate 변경 (ADR-PENDING-TM-70)**:
   - 단일 회차 평균이 아니라 **N=3 회차 평균 (또는 N=3 multi-judge 평균)** 으로 75 게이트 평가.
   - per-sample Δmax > 10 인 경우 별도 flag 로 표기.
   - 회차 간 비교 시 ±10 noise band 내 변화는 "유의미 회귀 아님" 으로 분류.
4. **(선택) 이중 judge / self-consistency**: gpt-4o 와 claude-sonnet 양쪽 호출 후 평균.
   비용 2배. judge variance 가 prompt-side 외 noise 라면 효과 큼.

- 시간: 약 25분 (코드 작성 10분 + 실험 ~2분 + 분석/문서 ~13분).

## 결론

**r3→r4 -7.8 회귀는 "회귀" 라기보다 측정 노이즈**다. judge config 의 결정성 결여가 dominant 원인이며,
이것이 해결되지 않은 상태에서 단일 회차 결과로 fix 의 효과/회귀 여부를 판단하면 false signal 에
끌려다니게 된다. `temperature=0 + seed` 가 즉시 적용 가능한 최소 변경이며, 더 나아가서는 N-shot
평균을 acceptance gate 로 박제할 필요가 있다.
