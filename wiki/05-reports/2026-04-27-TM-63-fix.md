# TM-63 — Constellation 별 분포 균일화

## 요약

`Constellation` 템플릿의 별 좌표 분포가 **한쪽(우측)에 쏠려** 좌측 상단이 sparse한 문제 (TM-43 시각 검수에서 발견)를 **Halton(2, 3) low-discrepancy sequence + sub-cell jitter** 로 수정.

- 파일: `src/remotion/templates/Constellation.tsx`
- 변경 라인: stars 좌표 생성 블록 (sin-hash → Halton + jitter)
- 정성 검증: dev 캡처에서 좌·우·상·하 모두 별 균일 분포 확인
- 정량 검증: 4분면 카운트 stddev `4.39 → 0.50` (약 9× 균일)

## 원인 분석

기존 코드는 별 좌표를 다음 sin-hash PRNG로 생성:

```js
const rnd = (k) => {
  const s = Math.sin(seed * 12.9898 + k * 78.233) * 43758.5453;
  return s - Math.floor(s);
};
const baseX = rnd(1) * width;
const baseY = rnd(2) * height;
```

이 PRNG는 GLSL 셰이더에서 흔히 쓰이는 단순 hash로, 작은 정수 seed (1..70) 입력 시 **출력 분포가 비균일**한 게 알려진 약점이다. 특히 N≈70 정도의 작은 표본에서 한쪽 사분면에 점이 쏠리는 빈도가 높다.

**4분면 분포(N=70, 1920x1080):**

| 코너 | 기존 (sin-hash) | 신규 (Halton + jitter) |
|---|---|---|
| top-left | 16 | 18 |
| top-right | 14 | 18 |
| bottom-left | 15 | 17 |
| bottom-right | 25 | 17 |
| **stddev** | **4.39** | **0.50** |

(브라우저 콘솔에서 동일 시드 입력 시 동일 결과 — 결정적, 재현 가능.)

## 해결책

**Halton(base=2, x) × Halton(base=3, y)** 시퀀스로 base 위치 생성. Halton sequence는 quasi-random low-discrepancy sequence로, 임의 N에 대해 [0,1]² 공간 균일 커버를 보장한다.

```js
const halton = (index, base) => {
  let f = 1, r = 0, i = index;
  while (i > 0) {
    f /= base;
    r += f * (i % base);
    i = Math.floor(i / base);
  }
  return r;
};
// 각 별마다 cell 단위 sub-cell jitter 추가 → 격자처럼 안 보임
const cell = 1 / Math.sqrt(count);
const jitterX = (rnd(7) - 0.5) * cell;
const jitterY = (rnd(8) - 0.5) * cell;
const ux = ((halton(seed, 2) + jitterX) % 1 + 1) % 1;
const uy = ((halton(seed, 3) + jitterY) % 1 + 1) % 1;
```

velocity / size / color / twinkle은 기존 sin-hash rnd를 그대로 사용 (시각 다양성 유지). drift wrap-around는 그대로.

### 왜 Poisson disk가 아닌가

- Poisson disk sampling은 매 프레임 O(N²) 거부 sampling — Remotion은 매 프레임 컴포넌트가 재실행되므로 비용이 부담.
- Halton은 O(N log N) (각 별 O(log seed)), 그리고 **결정적** — frame-stable.
- Jitter로 격자감 제거 → 시각적으로 충분히 자연스러움.
