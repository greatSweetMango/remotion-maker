# TM-86 — Retro

## TL;DR

Sequence-aware sidebar 라이브 동작 검증 PASS. spec 문구와 실제 ADR-0008 구현 범위 불일치 발견 — 별도 후속 분리 권장. Playwright MCP 가 worktree 간 공유되는 점이 새 gotcha.

## 잘 된 것

- Worktree + dev:3086 격리 setup 깔끔하게 동작 (`scripts/setup-worktree.sh`).
- Product Intro 60s 템플릿이 그 자체로 5-sequence·13-PARAMS multi-segment fixture 라 LLM 호출 없이 결정적 회귀 검증 달성. 비용 ~$0.
- Playwright `browser_evaluate` 안에서 한 번에 click + KeyboardEvent + PointerEvent drag 시퀀스를 묶어 cross-tab race 회피.
- panel resize min/maxSize clamp 가 코드(TM 'react-resizable-panels v4 string %' gotcha) 대로 정상 작동 — 회귀 없음.

## 안 된 것 / 어려웠던 것

- **Playwright MCP shared browser**: 다른 worktree agent (TM-79 on :3079) 가 같은 브라우저에 새 탭을 열어 현재 탭 포커스를 빼앗음. `browser_tabs select 0` 으로 회피했으나 매번 검증 필요.
- **Cookie leakage across ports**: NextAuth dev cookie 가 `127.0.0.1` host scope 라서 :3079 와 :3086 세션이 섞일 수 있음. NEXTAUTH_URL pin 만으로 충분치 않은 케이스 존재.
- **Spec drift**: TM-86 본문이 "drag-reorder / scene 분리·추가·삭제" 를 요구하나 ADR-0008 / 코드는 read-only filter. orchestrator/PM 단계에서 task description 과 ADR scope 정합성 점검 부족.

## 배운 점

- "시각적 검증" task 라도 LLM 호출이 본질이 아니라면 결정적 fixture(기존 multi-segment 템플릿) 활용이 더 신뢰성 높고 저렴.
- Playwright MCP 도 `browser_tabs list/select` 를 첫 액션으로 두면 cross-agent 충돌을 빠르게 잡아냄.
- `history.replaceState` 로 클라이언트 라우팅 재설정 하면 브라우저 navigation hooks 우회 가능 (cross-tab 충돌 회피용).
