# TM-102 — Multi-step generation pipeline

## TL;DR

Single-shot `generateAsset` plateaus at "thin" output (TM-46 r6: ~70 visual).
Added a 3-stage pipeline (outline → scene-spec → scene-code → compose)
behind feature flag `AI_MULTI_STEP=1`. Live smoke on 5 baseline prompts:
multi-step succeeds 5/5 (parity with single-shot), produces ~16% longer code
(2436 vs 2092 chars) at ~2.3× wall time (13.6s vs 6.0s). Off by default
until TM-46 r7 confirms a visual-judge uplift.

## 무엇이 바뀌었나

- New `src/lib/ai/pipeline.ts` exposing `generateOutline`, `generateSceneSpec`,
  `generateSceneCode`, `composeSceneCodes`, and `generateAssetMultiStep`.
- New prompts in `src/lib/ai/prompts.ts`: `OUTLINE_SYSTEM_PROMPT`,
  `SCENE_SPEC_SYSTEM_PROMPT`, `SCENE_CODE_SYSTEM_PROMPT` (each cacheable).
- `src/lib/ai/generate.ts` opts into the pipeline when `AI_MULTI_STEP=1`
  (no clarify answers in flight).
- `[[01-pm/decisions/0020-multi-step-pipeline|ADR-0020]]` records the
  rationale, cost tradeoff (1.4–2.4× tokens), and rollout phases.
- 14 new unit tests in `__tests__/lib/ai/pipeline.test.ts` (all passing,
  pre-existing `generate.test.ts` 19/19 still green).
- Gated live test `__tests__/lib/ai/tm-102-live-baseline.test.ts`
  (`TM102_LIVE=1`, ~$2 / run).

## 왜 / 배경

User feedback: "결과물 너무 부실. Claude artifact 수준이어야". Forcing
the model to plan + design + code in one breath consistently produces
under-specified scenes (single bar, missing labels, palette drift).
Splitting the work mirrors how Artifacts-grade output is produced in
practice: plan → flesh out → render.

## 영향

- **코드/시스템**: opt-in feature flag, no production traffic until
  benched. Keeps ADR-0001/0002/0003 intact (the composed code still
  exports a top-level `PARAMS` const and keeps `cache_control` on each
  stage's system prompt).
- **사용자/제품**: when flag is on, output gains structured per-scene
  PARAMS (`scene1_*`, `scene2_*`) + a top-level `<Sequence>` timeline.
- **비용/성능**: `MULTI_STEP_COST_RATIO_WARN = 1.7` triggers a runtime
  warning when projected token-multiplier crosses the threshold. Live
  run measured ~2.3× wall ms on Sonnet — explicit user opt-in needed
  before flipping default-on.

### Live smoke summary (Anthropic Sonnet, 5 prompts)

| mode        | ok / 5 | mean code len | mean wall ms |
|-------------|--------|---------------|--------------|
| single-shot | 5      | 2092          | 6023         |
| multi-step  | 5      | 2436          | 13651        |

Code-length isn't the visual-judge proxy we'd actually decide on —
TM-46 r7 will run the rubric over both modes on the same prompts.
