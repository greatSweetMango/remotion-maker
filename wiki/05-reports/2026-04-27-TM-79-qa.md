## Findings (actionable)

### F1 — Playback rate (speed) control is missing from studio Player [bug or by-design?]

- The Remotion Player supports `playbackRate` and exposes a popup via `showPlaybackRateControl` + `playbackRates` props. PlayerPanel sets neither.
- Users who expect "video speed 0.5×/1×/2×" cannot change it from the studio. The visible "Speed" slider belongs to the template parameter system (e.g. counter increment speed), and binding the playback rate UI to it would conflate semantics.
- **Recommendation**: TM-79 spec says "QA pause/seek/loop/**speed**" — if "speed" means video playback rate, this is a missing-feature finding. Spawn AI-BUG-PLAYER-no-playback-rate (priority: low; UX gap, not a regression).

### F2 — Cross-worktree port collision blocks Playwright QA reliability [infra/env]

- Multiple worktrees (TM-74/79/81/84) run dev servers concurrently on different ports. During the QA session, page navigation initiated on `:3079` repeatedly snapped to `:3084` (a different running worktree) — likely because Studio links/TemplatePicker render absolute URLs derived from a stale environment, OR because the browser context retained a session cookie scoped to a sibling port and a soft-redirect chain crossed origins.
- This is repro: `goto :3079/studio?template=counter-animation` → wait 3s → URL becomes `:3084/studio?template=<other>`.
- **Recommendation**: orthogonal to TM-79. File AI-INFRA-worktree-port-bleed for investigation. Suspect candidates: `NEXTAUTH_URL` mis-pinning in some sibling worktree, or marketing carousel redirect logic.

### F3 — Loop works but invisible [doc gap, low]

- `loop={true}` is hardcoded. Users have no toggle. Acceptable for an editor preview, but worth mentioning in user-facing docs that "preview always loops; export uses one-shot".
