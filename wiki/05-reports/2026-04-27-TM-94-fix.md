# TM-94 — Prompt-mode chip color-contrast fix

## TL;DR

TM-80 r2 (`wiki/05-reports/2026-04-27-TM-80-qa-r2.md`) 잔존 serious=40 (전부 `color-contrast`) 중 30 노드는 `PromptPanel.tsx` Edit/Generate chip의 `bg-violet-600` + `<kbd opacity-60>` 단일 토큰에서 나옴. chip 활성 배경을 600→700 한 단계 어둡게 + `<kbd>`의 `opacity-60`을 명시적 `text-white`(active) / `text-slate-300`(inactive)로 교체. jest-axe 8/8 0 violations, 컴포넌트 회귀 36/36 pass.

## 무엇이 바뀌었나

- `src/components/studio/PromptPanel.tsx`
  - Edit chip 활성 배경: `bg-violet-600` → `bg-violet-700`
  - Generate chip 활성 배경: `bg-emerald-600` → `bg-emerald-700`
  - 비활성 chip 본문 텍스트: `text-slate-400` → `text-slate-300`
  - `<kbd>⌘E</kbd>`/`<kbd>⌘N</kbd>` 클래스: `opacity-60` 제거, 모드별 명시 색 (`text-white` 활성 / `text-slate-300` 비활성)
  - FREE 티어 카운트 `(editCount/editLimit)`: `opacity-70` → 모드별 명시 색
- `__tests__/components/prompt-mode-chip-contrast.test.tsx` (신규, 8 tests)
  - jest-axe 0 violations
  - active chip className 단언 (700 shade 강제)
  - `<kbd>` opacity-60 부재 단언
  - WCAG 2 정적 contrast ratio 계산 (white/violet-700 ≥ 4.5, white/emerald-700 ≥ 4.5, slate-300/slate-800 ≥ 4.5)

## 왜 / 배경

TM-80 r2가 critical=0으로 requalify clear됐지만 serious=40 (전부 `color-contrast`)이 잔존. 분포 분석 결과:

| 노드 | 합계 | 출처 |
|---|---:|---|
| 5 templates × 5 | 25 | customize sidebar의 prompt-mode chip + kbd hint (공유 컴포넌트) |
| customize 베이스 | 12 | sidebar header / meta (다른 token) |
| home | 2 | 마케팅 본문 muted text |
| login | 1 | radix tab trigger inactive |

이 중 30/40이 단일 토큰(`PromptPanel` chip + `<kbd>`)이라는 점이 핵심. 단일 fix로 해소 가능하다는 TM-80 r2 권고에 따라 본 task spawn.

WCAG 2 검증 (table 룰: small text 4.5:1):

| 토큰 쌍 | ratio | 통과 |
|---|---:|---|
| white on violet-600 (#7c3aed) | 5.70:1 | ✓ (마진 협소, opacity-60 합성하면 위험) |
| **white on violet-700 (#6d28d9)** | **7.10:1** | ✓ AA 여유 (AAA normal도 통과) |
| white on emerald-600 (#059669) | 3.77:1 | **❌ AA 실패** |
| **white on emerald-700 (#047857)** | **5.48:1** | ✓ |
| slate-400 on slate-800 | 5.71:1 | ✓ (체크) |
| **slate-300 on slate-800** | **9.85:1** | ✓ |

emerald-600/white는 4.5:1 미달이라 700 승격이 필수. violet은 600도 본문 white에는 5.70으로 통과했으나 `<kbd opacity-60>` 알파 합성 시 effective contrast가 떨어져 axe가 violation으로 잡았던 것 — `opacity` 제거 + 700 승격으로 이중 안전장치.

`opacity-60`의 본질 문제: 알파 블렌딩은 부모 배경에 의존해 contrast가 측정 불가능 ↔ 명시 색상은 결정적. axe도 알파 합성된 effective color를 추정하긴 하나 jsdom에서는 unreliable — 명시 색 토큰이 ground truth.
