# TM-43 회고 — 35개 템플릿 시각 검수

## What worked
- Playwright MCP로 35개 템플릿 순회 + 105 스크린샷 캡처 자동화. OpenAI 호출 0회로 비용 $0.
- 패턴: navigate → wait 2.5s → start screenshot → wait 1s → mid screenshot → wait 1s → end screenshot → 다음 navigate. 안정적.
- Studio 페이지의 `?template=<id>` 라우팅이 모든 35 템플릿에서 server-side로 initialAsset 주입 정상.

## What didn't work (and how we recovered)
1. **Multi-worktree origin 충돌** — 다른 worktree dev 서버들이 같은 `localhost`에서 cookie 공유 → 우리 페이지가 다른 포트로 점프.
   - 해결: `127.0.0.1` 호스트 + `allowedDevOrigins` 추가 + NEXTAUTH_URL 변경.
2. **wait_for 동안 페이지가 외부 origin으로 navigate** — 이는 #1의 결과. 격리 후 사라짐.
3. **dev server가 중간에 종료** — 한 차례 발생, 재시작으로 회복.
