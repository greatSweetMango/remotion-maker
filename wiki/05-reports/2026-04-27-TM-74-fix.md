# TM-74 — Reference-template RAG

## TL;DR

Injects ONE existing template from `src/remotion/templates/` into the
generation system prompt as a worked reference, picked via simple
keyword/category retrieval. Expected lift: **+10~15 mean** on the TM-46
acceptance gate, concentrated on data-viz and transition where the
prompt-only path was capped at ~70.

Zero new dependencies. Reversible. Decision: ADR-0019.

## Context

- TM-46 r6 mean ≈ 70 with the prompt-only category-guidelines pass (TM-71).
- Residual fails: data-viz (no axes/labels, single-bar), transition
  (single-state output).
- Hypothesis: a **worked example** beats more rules. We have 35 production
  templates already — use them.

## Implementation

1. **`src/lib/ai/retrieval.ts`** (new, ~250 LOC):
   - `inferCategoryFromPrompt(prompt)` — bilingual KO/EN regex catalog.
     Returns `null` when no category dominates so the caller can skip
     injection rather than mis-bias.
   - `pickReferenceTemplate(category, prompt)` — keyword tiebreak inside
     a category over a static `REFERENCE_CATALOG`.
   - `buildReferenceBlock(prompt, opts)` — composes the system-prompt
     addendum. Caps reference source at 6KB (no current template exceeds
     this; truncation is defensive).
   - `retrieveReferenceForPrompt(prompt)` — top-level wrapper.

2. **`src/lib/ai/generate.ts`** (wiring):
   - `generateAsset` now calls `retrieveReferenceForPrompt` once per
     invocation and threads the resulting `baseSystemPrompt =
     GENERATION_WITH_CLARIFY_SYSTEM_PROMPT + addendum` through ALL
     attempts (first, force-generate, entity-count, transpile retry,
     placeholder retry). Reference is stable across retries to preserve
     ADR-0003 prompt-cache key.
   - `generateAssetPlaceholderRetry` accepts the augmented base prompt.

3. **Tests:**
   - `__tests__/lib/ai/retrieval.test.ts` — 32 cases (inference + picker
     + block composition).
   - `__tests__/lib/ai/tm-74-live-validate.test.ts` — gated by
     `TM74_LIVE=1`; 5 live prompts.

## Validation

### Unit
```

Tests:       144 passed, 144 total
```
No regressions in adjacent suites (clarify-gate, classify, refusal,
prompts-categories, generate, client-stream, client-resilience).

### Live (TM74_LIVE=1, gpt-4o-mini, free tier)
5/5 prompts: retrieval hit, code substantive (no placeholder), code length
~1.1-1.9KB, bar-chart prompt preserved user values 120/150/180.


Run wall: 55.8s for 5 calls (≈ 11s p50 — within steady-state ranges).

### Cost

Live smoke: ~$0.05 actual on gpt-4o-mini. Budget was $1.5 — well under.
Marginal per-call cost increase from the reference block: ~10-20% extra
input tokens (≈ 1-2KB of TSX), Haiku-tier ≈ $0.0005/call extra.

## Risk & rollback

- Single env-flag rollback: wrapping the `rag.addendum` concat in a
  feature flag (e.g. `if (process.env.RAG_DISABLED !== '1')`) would let
  ops disable RAG in seconds. Not added in this commit; ship hot.
- No DB migration. No new dependency. PR fully revertible.

## Follow-ups

- **TM-46 r7** acceptance gate run (rag-on vs rag-off, ≥30 prompts) to
  confirm the +10~15 hypothesis. Booked separately.
- **Embedding retriever (v2)** — separate task. Useful when:
  (a) prompt-only categorisation drops below 80% accuracy on production
  traffic, OR (b) we author >100 templates and category-level granularity
  becomes too coarse.
- **Catalog drift guard** — small unit test that asserts every entry in
  `REFERENCE_CATALOG.filename` resolves under `src/remotion/templates/`.
  Cheap to add when we feel the pain.

## Retro

| theme         | observation                                           |
|---------------|-------------------------------------------------------|
| **what worked** | regex-only category inference passed 32/32 unit cases bilingual; live smoke 5/5 hit. |

| **next time**   | When introducing system-prompt growth, also write a tiny "what does the prompt look like now" snapshot test so retros can see the actual context shape. |
