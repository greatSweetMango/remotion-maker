# TM-82 retro

## What we did

## What worked

- **Failure injection at the SDK seam** rather than HTTP. Lets us

  `prisma.user.updateMany` + `update.decrement` â€” let the refund
  invariant be expressed as "count(decrement calls) == count(reserve
  calls)" which is independent of error class.
- **Explicit retry button > automatic backoff loop**. Found in the
  current code that both SDKs already have built-in retries, so adding
  ours would 4â€“6Ă— the cost on outages. Manual retry is correct here.
