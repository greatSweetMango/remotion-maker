# TM-41 — AI 생성 플로우 E2E QA 보고서

## TL;DR

- 60개 호출 (FREE 50 + PRO stratified 10)을 `generateAsset()` 직접 호출로 측정.
- **통과율 90.0%** (acceptance ≥95% 미달, **5점 미달**).
- **첫 프레임 p50 = 10,096 ms** (acceptance < 5,000 ms 의 **2배 초과**).
- **콘솔 에러 0** acceptance 위반 — 홈/대시보드 모든 페이지에서 CSP `media-src` 위반 175건+ 반복.
- evaluator 통과율은 sandbox `validateCode` 기준 PRO 4건이 spec 위반 (PARAMS 누락 + 25자 stub code).
- **VERDICT: REQUEST_CHANGES** — 5건의 fix task spawn (TM-50~54), 모두 `triggers_requalify=[41]`.

## 측정 메소드

- 입력 프롬프트: `__tests__/benchmarks/params-extraction.benchmark.ts` 의 `BENCHMARK_PROMPTS` 50개 (data-viz 10, text-anim 10, transition 10, loader 10, infographic 10; ko/en 혼합).
- 매트릭스: FREE 50 (gpt-4o-mini) + PRO 10 (gpt-4o, 카테고리당 2개 stratified sample).
- 인증 우회: `/api/generate` 의 thin wrapper 를 건너뛰고 `generateAsset(prompt, model)` 를 in-process 호출. DB write 만 누락 — 응답 구조/PARAMS/evaluator/시간 측정에는 영향 없음.
- 첫 프레임 시간: `generateAsset` 시작 ~ asset 반환까지 wall ms.
- evaluator 친화도: 응답 코드의 `validateCode` 통과 + transpileTSX 성공 + `extractParameters` ≥ 1.
- 시각 캡처: dev 서버(`localhost:3041`) Playwright MCP 로 customize 페이지 + dashboard + 홈 1프레임씩 캡처. 세부 60프레임 동영상 캡처는 시간 예산상 1프레임씩 대체.
- 결과 raw: `__tests__/qa/results/tm-41-results.json`.

## 추가 발견 (시각 캡처 부산물)

- 홈/대시보드/스튜디오 모든 페이지에서 **CSP `media-src` 위반 175+회**. `data:audio/mp3;base64,…` 가 정책 `media-src 'self' blob: https:` 에 차단됨. `counter-animation` 템플릿의 sound effect 가 inline data URI 로 박혀있는 것으로 추정. → TM-50.
