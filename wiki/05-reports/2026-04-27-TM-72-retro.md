# TM-72 — Retrospective

## 결과 요약

- **Status**: completed
- **Code 변경**: `src/lib/ai/client.ts` (+~50 lines), `__tests__/lib/ai/client-stream.test.ts` (+95 lines, 4 tests), `scripts/qa/tm-72-capture-variance.mjs` (신규, ~180 lines).
- **테스트**: client-stream.test.ts 11/11 pass (기존 7 + 신규 4).
- **라이브 검증**: 30 OpenAI 호출, paramsΔmax mean 49 → 7 (−86%), codeΔmax mean 615 → 312 (−49%). 자세한 수치는 `2026-04-27-TM-72-fix.md`.
- **비용**: ~$0.5 OpenAI (gpt-4o-mini × 30 calls).

## 잘된 점

1. **TM-70 패턴 재사용**: judge-side 결정성에 썼던 정확히 같은 fix (`temperature=0, seed=42`) 를 capture-side 에 옮긴 것이라, 디자인 결정/리스크 평가 부담이 없었음.
2. **env 으로 override** 가능하게 만들어서 향후 A/B 실험 / 사용자 의도적 stochastic 호출 시 코드 수정 없이 가능.
3. **측정 우선** — 이론적 주장이 아니라 30 호출 라이브 데이터로 효과 입증. paramsΔmax 86% 감소는 acceptance-gate 신뢰도에 직접 기여.

## 아쉬운 점

1. **`seed` 의 best-effort 한계** — P2 (긴 free-form) 는 여전히 1210 chars 차이. 솔직한 한계로 보고에 박제. 완전한 결정성은 별도 task (prompt-hash cache layer) 가 필요함.
2. **Anthropic 측에 seed 미적용** — SDK 가 받지 않음. 대신 temperature 만 적용 — 향후 SDK 업데이트 시 즉시 추가 가능하게 주석 박제.
3. **검증 스크립트가 production code path 를 100% 재현하지 않음** — system prompt 를 단순화한 버전 사용. 풀 시스템 프롬프트 + prompt cache 가 결과 분포를 약간 다르게 만들 수 있음. 다만 본 task 의 목표 (parameter forwarding 검증 + 효과 크기 추정) 에는 충분.

## 학습

- OpenAI `seed` 는 짧은 구조화 응답 (JSON envelope, PARAMS block) 에는 매우 잘 붙고, 긴 free-form prose/code 에서는 약함. **"PARAMS 만 안정화하고 코드 본문 다양성은 허용"** 이 현재 도달 가능한 최선.
- TM-70 처럼 "judge 결정성" 만 잡으면 acceptance-gate noise 의 절반만 잡힌다. capture-side 도 같이 잡아야 회차 간 비교가 의미를 가짐.
- `cache_control: ephemeral` (ADR-0003) 와 `temperature=0+seed` 는 직교 — 둘 다 켜는 게 안전함 (cache 는 latency, seed 는 determinism).

## triggers_requalify

- 다음 iter 의 TM-46 r6 owner 는 동일 30 prompt 로 **새 capture config + 새 judge config** 둘 다 켜고 재캡처/재채점. r5 와의 차이가 통계적 유의 수준에 도달하면 비로소 "TM-67/68/69 효과 측정" 이 가능해짐.

## 후속 권고

1. (이미 별도 task 로 권고) **N-shot 평균 acceptance gate** — TM-70 retro 의 권고와 묶어서 한 번에 도입 권장.
2. (선택) **prompt → asset cache layer** — 동일 prompt + 동일 model + 동일 seed 면 LLM 호출을 건너뛰고 이전 결과 재사용. 비용 + UX 모두 이득. 별도 task 필요.
3. Anthropic SDK 의 `seed` 지원 추적 — 추가되면 5 줄 변경으로 양쪽 provider 결정성 통일.
