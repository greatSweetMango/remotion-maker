# TM-113 — setup-worktree.sh 재실행 시 .env.local 보존

## TL;DR

`scripts/setup-worktree.sh` 재실행 시 main의 `.env.local`을 통째로 덮어쓰던 동작을 제거. 이제 worktree에 이미 `.env.local`이 있으면 그 파일을 보존하고 `NEXTAUTH_URL`/`DATABASE_URL` 두 줄만 in-place로 갱신한다. 기존 NEXTAUTH_URL 동적 치환(host+port 인자) 로직은 이미 정상 동작하고 있었음.

## 무엇이 바뀌었나

- `scripts/setup-worktree.sh`
  - Step 1 (`cp main/.env.local → worktree/.env.local`)을 **존재하지 않을 때만** 실행하도록 가드 추가.
  - 재실행 시 worktree의 기존 `.env.local`을 그대로 두고 `awk` 치환만 수행 → 사용자가 추가한 worktree-local 환경 변수(예: 별도 OPENAI 키, 디버그 플래그)가 보존됨.
  - 헤더 주석/완료 로그 문구를 멱등성 의미에 맞게 수정.

## 왜 / 배경

- TM-112 작업 중 발견: setup-worktree를 두 번째 돌리면 `.env.local`이 main 사본으로 reset되어 매번 worktree-specific 값(예: 다른 PORT, 추가 시크릿)을 다시 손으로 넣어야 했음.
- 원 spec(TM-113)은 "NEXTAUTH_URL 자동 치환"이었으나 코드 검토 결과 동적 치환은 이미 구현되어 있었고(line 109-127 awk), 진짜 페인은 **재실행 멱등성** 쪽이었다.
- 멱등성의 정의를 "source-replacing(매번 main 복제)"에서 "value-preserving(이미 있는 값 유지 + 관리 라인만 갱신)"으로 재정렬.

## 영향

- 코드: `scripts/setup-worktree.sh` 헤더 주석 + Step 1 분기 + 로그 메시지.
- 사용자: 본 worktree 부트스트랩 검증 (port 3113) — `HTTP 200` 정상.
- 멱등성 검증:
  - `TM113_TEST_FLAG=preserved-by-idempotency` 추가 후 재실행 → 라인 보존 확인.
  - port 9999로 재실행 → `NEXTAUTH_URL`만 갱신, 사용자 라인 유지.
  - port 3113으로 reset → 정상 복귀.
- 비용/성능: 변화 없음. OpenAI 호출 0.
