# TM-59 — D1-D7 사용자 향 거절 메시지 카테고리화

## 한줄 요약

TM-45 fuzz D1-D7 (jailbreak / unsafe-code injection prompts) 가 LLM 자체 거절로 인해 "AI did not return valid JSON" 으로 표면화되던 문제를 해결. `src/lib/ai/refusal.ts` 분류기를 추가해 raw 응답을 **safety / adversarial / policy / unknown** 4 카테고리로 분류, 카테고리별 한국어 메시지와 stable error code (`AI_REFUSAL_*`) 로 응답. `/api/generate` 는 거절을 400 (사용자 입력 문제) 으로 반환하며 monthly quota 를 차감하지 않는다.

## 배경

TM-45 fuzz 30 케이스 중 D1-D7 8 케이스가 `<script>`, `eval`, `document.cookie`, `new Worker` 등 unsafe-code 임베드를 요청. LLM 자체 정책으로 거절 → `extractJson` 이 prose 를 파싱 못 함 → `throw new Error('AI did not return valid JSON')` → 5xx + 모호한 toast.

문제: 사용자가 **왜** 거절됐는지 단서가 없어 같은 프롬프트를 반복 시도하거나 본인 프롬프트가 합법인데 모델 결함으로 오인. TM-45 follow-up TM-57/58 머지 후 유일하게 남은 입력 검증 갭.

## 변경

### `src/lib/ai/refusal.ts` (NEW)

- `RefusalCategory = 'safety' | 'adversarial' | 'policy' | 'unknown'`
- `classifyRefusal(rawText)` — 영/한 패턴 휴리스틱:
  - **adversarial**: `ignore previous instructions`, `override system`, `reveal/leak/disclose ... prompt|instructions`, `jailbreak`, `prompt injection`, `시스템 ... (무시|변경|덮어쓰)`
  - **safety**: `content/usage/safety polic(y|ies)`, `I('m| am) sorry ... I (can't|cannot)`, `I (can't|cannot|won't|will not) (help|assist|comply|provide|generate|create)`, harm keywords (violence/hate/sexual/self-harm/...), `(폭력|혐오|성적|자해|불법|증오)`, `(안전|콘텐츠|이용)\s*정책`, `(도와드릴|처리할|...)\s*수\s*없`
  - **policy**: 기타 `I (can't|cannot)`, sorry, `(죄송|미안)`
  - **unknown**: 위 어느 패턴에도 매치 안 됨 → 기존 "AI did not return valid JSON" 으로 fallback (실제 파서 실패 보존)
- 순서: **adversarial 먼저 → safety → policy** (jailbreak 의도가 더 구체적이므로)
- `refusalMessage(category)` — 한국어 toast 메시지:
  - safety: "이 요청은 안전 정책상 처리하기 어렵습니다. 프롬프트를 다시 작성해 주세요."
  - adversarial: "프롬프트가 시스템 지시를 변경하려 하기 때문에 처리할 수 없습니다. 일반적인 영상 설명으로 다시 작성해 주세요."
  - policy: "이 요청은 처리할 수 없습니다. 프롬프트를 다시 작성해 주세요."
  - unknown: "AI 응답을 해석하지 못했습니다. 잠시 후 다시 시도해 주세요."
- `refusalCode(category)` — `AI_REFUSAL_SAFETY | AI_REFUSAL_ADVERSARIAL | AI_REFUSAL_POLICY | AI_PARSE_FAILED`
- `AiRefusalError` — `category` / `code` / `matchedHint` 동반 Error 서브클래스

### `src/lib/ai/generate.ts`

`extractJson(text) === null` 분기에서:

```ts
const classification = classifyRefusal(text);
if (classification.category !== 'unknown') {
  throw new AiRefusalError(classification);
}
throw new Error('AI did not return valid JSON');
```

기존 코드 흐름은 그대로 유지. `unknown` 일 땐 legacy 메시지 그대로 표면화 (실제 파서 버그 케이스 디버깅 가능성 보존).

### `src/app/api/generate/route.ts`

`catch` 블록 첫줄에서 `AiRefusalError` instanceof 체크:

```ts
if (error instanceof AiRefusalError) {
  console.warn(`[generate] refusal category=${error.category} hint=${error.matchedHint ?? '-'} tier=${user.tier}`);
  return NextResponse.json(
    { error: error.message, code: error.code, category: error.category },
    { status: 400 }, // 사용자 입력 문제 — quota 미차감
  );
}
```

400 으로 반환 이유: 거절은 **사용자 입력의 문제** (정책 위반) 이며 서버 5xx 가 아니다. 또한 monthly quota 가 LLM 호출 후 `prisma.user.update increment` 로 차감되는데, 이 분기는 그 라인보다 앞에서 throw 되므로 자연히 무차감.

## 정보 노출 정책

거절 메시지는 카테고리만 알리고 **deny-list 패턴 / 시스템 프롬프트 / 매치된 hint 는 노출 X**.
- `matchedHint` 는 `console.warn` 서버 로그에만 (운영/디버그용).
- `AiRefusalError.message` 는 사용자용 — hint 미포함, 단위 테스트로 강제 (`__tests__/lib/ai/refusal.test.ts: does not leak the matched hint`).
- "Forbidden: Worker" 같은 sandbox-detail 메시지는 **본 변경 범위 밖** (별도 sandbox 계층). 본 PR 은 LLM 자체 거절만 다룬다.

## 검증 / 한계

- 휴리스틱 기반 — false positive (실제 파서 버그를 거절로 오분류) 시 사용자 안내가 어긋날 수 있음. 이를 방지하기 위해 `unknown` 폴백을 의도적으로 보수적으로 유지.
- 휴리스틱은 영/한 위주. 모델이 다른 언어로 거절할 경우 `unknown` 으로 떨어져 legacy 메시지가 표면화됨 — 안전한 기본값.
- 모델 응답이 부분 JSON 을 emit 하면서 그 안에 거절 prose 를 포함할 수도 있음 — 현재는 `extractJson` 이 성공하면 통과시키므로 본 분류 경로 미진입. 향후 회귀 발견 시 별도 task.
