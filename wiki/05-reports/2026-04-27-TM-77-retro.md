# TM-77 회고

## 잘된 점

- ADR-0006을 빠르게 다시 읽음으로써 hook이 "Claude Code 자기 호출"만 본다는
  암묵적 가정을 즉시 식별. 앱-사이드 sink 추가로 24시간 자율 예산 캡 자동화의
  마지막 누락을 메움.
- `SPEND_FILE_PATH` env 오버라이드를 처음부터 둬서 단위 테스트가 실제 production
  파일에 손대지 않고도 race-free 시나리오까지 검증.
- OpenAI streaming의 `stream_options.include_usage`를 적시에 적용 — 그렇지 않으면
  스트리밍 응답에서 usage가 null로 떨어진다.

## 아쉬운 점 / 학습

- jq가 사용자 환경에 설치돼 있지 않음 → bash hook 자체는 사용자 머신에서
  무음 no-op 상태였을 가능성. 앱-사이드 추적이 갖춰졌으니 직접적 영향은 적지만,
  운영 README에 `brew install jq` 안내가 누락된 점은 따로 보강 필요.
- ADR-0006이 "PostToolUse만으로 충분하다"는 식으로 적혀 있어 후속 ADR
  (앱-사이드 sink) 발행 여부 검토 필요. 현재는 ADR-0006 보강만으로 충분하다고
  판단 (대안 분기에 같은 사고 흐름 이미 기록됨).
