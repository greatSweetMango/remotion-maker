# TM-46 r6 회고

## 잘 된 점

1. **capture+judge 양면 결정성 검증** — TM-70 (judge) + TM-72 (capture) 합산 효과를
   N=2 데이터로 측정. round-level std 가 r5 (≈14.8) → r6 1.77 로 약 88% 감소.
   ADR-0016 C2 (std<5) 첫 PASS.
2. **per-prompt mean_std 33% 감소** (r5 8.10 → r6 5.33). std=0 인 prompt 16.7% → 40%
   로 증가. 결정성 측면의 진보가 데이터로 명확.
3. **r4→r5 의 "noise vs 회귀" 가설을 r6 가 깔끔하게 정리** — r6 의 round std 1.77 은
   r3 71.2 / r4 63.4 / r5 67.8 / r6 68.65 의 회차 차이 (8 점대) 중 r5→r6 (0.85) 만
   진짜 변화이고 나머지는 noise였음을 retroactively 확인.
4. **ADR-0016 4 기준 산식이 의사결정에 직접 도움** — C2 PASS / C1·C3·C4 FAIL 분리
   덕분에 "결정성은 잡혔지만 품질은 잡히지 않았다" 라는 명확한 결론. acceptance v1 의
   "MISS" 단일 비트로는 표현 불가능했던 정보가 4 기준에서 살아남.
5. **acceptance gate 정의의 검증 완료** — ADR-0016 의 noise band (5점) 가 측정도구
   결정성과 정합됨을 r6 round std 1.77 로 확인. gate 자체 재정의 불필요.

## 아쉬운 점

1. **acceptance MISS 5회차 연속 (C1)** — mean 자체가 75 미달 (68.65). prompt-only 한계.
   reference 템플릿 도입 없이는 추가 회차 ROI 0.
2. **dv-08 max_std=45.96 (collapse)** — capture 결정성 도입 후에도 한 prompt 가 회차
   간 65점 차이. 5+ entity prompt 가 max_tokens 근접해 truncation 가능성 — TM-72 의
   temp=0/seed 만으로는 부족함이 데이터로 확인됨.
3. **워크트리 prisma client isolate 부재** — dev 서버가 main 리포의 dev.db 를 열고
   있어 generate 200 limit 을 main user 가 hit. usage cap 임시 상향으로 우회. 별도
   infra task 필요 (TM-NN AI-INFRA-worktree-prisma-isolate).
4. **transition 카테고리 미해결** (55.3) — r3~r6 내내 60 미달. tr-03 / tr-08 양 회차
   동일 38 점. RGB-split / iris-wipe 같은 효과는 prompt-only 로 안정적 출력 불가능
   하다는 정성적 결론. reference 템플릿 follow-up 필요.
5. **N=2 의 통계적 한계** — round-level std 에 대해 정량 추정엔 N≥3 필요. r6 의
   1.77 도 점추정. 향후 N=3 으로 확장 시 std 추정 신뢰도 향상.
