# TM-91 — Retro

## TL;DR

reducer 단순 확장이라 빠르게 구현. 8개 단위 테스트 한 번에 통과, 기존 customize-roundtrip 회귀 없음. depth cap / no-op 가드 / asset 전환 시 reset 같은 엣지케이스를 처음부터 명시했더니 추후 손볼 게 없었다.

## 잘된 것

- 리듀서 패턴 유지(전면 재구조화 X). `paramValues`는 그대로 top-level에 두고 history만 곁들였다 → 모든 기존 selector/consumer 변경 0.
- `studioReducer` + `initialState` + `HISTORY_DEPTH` export 해서 reducer-level 단위 테스트로 빠르게 검증. React env 띄울 필요 없음.
- 같은 값 쓰기 no-op: 컨트롤드 인풋의 무해한 리렌더가 history를 더럽히는 것 방지. 처음부터 박아둠.
- INPUT/TEXTAREA 포커스 시 단축키 패스스루: 텍스트 파라미터(예: title) 편집 중 Cmd+Z가 브라우저 텍스트 undo 대신 슬라이더를 되돌리는 어색함 회피.

## 아쉬운 것

- `useStudio`는 본래 hook이라 reducer 단독 export가 module-internal pattern을 깨뜨림. test 한정으로 받아들였지만 큰 hook이 되면 분리 필요.
- mobile/desktop 두 군데에 같은 props 4개를 전달 — Studio 컴포넌트가 좀 자라고 있음. 차후 컨텍스트로 빼는 것 검토.
