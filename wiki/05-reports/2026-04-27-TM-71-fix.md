# TM-71 — GENERATION_SYSTEM_PROMPT 카테고리 가이드라인 보강

## TL;DR

- TM-46 r3 에서 평균 71.2 (acceptance ≥ 75 미달) 의 주범인 **data-viz (44.5)** /
  **transition (63.5)** 카테고리를 겨냥해 `GENERATION_SYSTEM_PROMPT` 에
  카테고리별 명시 가이드라인 블록 (DATA-VIZ / TRANSITION / TEXT-ANIM /
  INFOGRAPHIC / LOADER) 을 추가.
- 5개 worst-case (dv-02, dv-03, dv-08, tr-02, tr-08) 직접 호출 검증:
  생성 코드가 실제로 다중 차트 요소·레이블·색상 보간(interpolateColors)·
  두 상태 보간을 포함하기 시작 — r3 회귀 원인이 prompt 수준에서 해소됨을 확인.
- TM-46 r4 에서 acceptance 재측정 예정 (`triggers_requalify=[46]`).

## 무엇이 바뀌었나

- `src/lib/ai/prompts.ts`
  - `GENERATION_SYSTEM_PROMPT` 끝에 **CATEGORY-SPECIFIC GUIDELINES** 블록 추가.
    DATA-VIZ: 데이터 값 모두 시각화 / 축·레이블 / 팔레트 hint 일관성 / 모션 패턴.
    TRANSITION: before+after 두 상태 필수 / midpoint 프레임 검증 / clip-path,
    interpolateColors, RGB-split 같은 구체 기술.
    TEXT-ANIM: typography first (실제 폰트/굵기/대비), 모션은 보조, Sequence 로
    멀티-스텝 명확.
    INFOGRAPHIC / LOADER: entry sequence / 주기적 모션.
  - `GENERATION_WITH_CLARIFY_SYSTEM_PROMPT` 는 `GENERATION_SYSTEM_PROMPT` 를
    concat 하므로 자동 상속.
- `__tests__/lib/ai/prompts-categories.test.ts` 신규
  - 각 카테고리 가이드라인 키워드/패턴이 prompt 에 박혀 있는지 jest 로 lock-in.
  - 5 cases all green.
- `__tests__/benchmarks/tm-71-validate.ts` 신규 (offline heuristic 검증)
  - 5 worst-case 를 `generateAsset` 로 직접 호출해 코드 길이 / PARAMS 개수 /
    JSX tag / 카테고리 motif (multipleElements, hasLabels, twoStates 등) 측정.
  - 결과: `__tests__/benchmarks/results/tm-71/validate.json` + `code/*.tsx`.

## 왜 / 배경

- TM-46 r3 보고 ([[2026-04-27-TM-46-visual-judge-r3]]) 에서:
  - data-viz 평균 44.5 (dv-02=15, dv-03=23 등). 한국어 + 숫자 배열 + 색상 톤
    복합 prompt 에서 generate 가 placeholder 에 가까운 컴포넌트만 출력.
  - transition 평균 63.5. tr-02 (검정→흰색 페이드) 43점은 **색상 보간 자체가
    미발생**. 모델이 단일 상태/freeze 만 그림.
- r3 retro 에서 prompt 보강 명령은 박혔지만 실제 task 가 없었음 — 본 task TM-71
  로 그 명령을 실행했다.
- 모델이 카테고리별로 무엇을 그려야 "충분히 좋은" 결과인지 모르는 게 본질
  원인 — examples 보다 explicit guideline 이 prompt cache 효율도 높다.

## 영향

### 코드 / 시스템 영향
- prompt 길이: ~3.3KB → ~5.3KB (+~2KB). Anthropic prompt-caching ephemeral
  적용 중이므로 후속 호출은 캐시 히트, 실제 토큰 비용은 거의 무영향.
- generate / clarify-generate 양쪽 경로에 동일하게 적용.

### 검증 (offline)
- 5/5 케이스 `generateAsset` 호출 성공 (1차 run).
- 2차 run 에서 stochastic 으로 dv-02 (security guard `require`) / tr-08
  (transpile retry 실패) 1회씩 실패 — prompt 변경과 무관한 기존 알려진 모드.
- dv-03 결과 코드: 4 slices (40/30/20/10) 각 색상 / svg path 분리 / label
  포함. r3 의 23점 결과와 대비해 명확한 개선.
- tr-02 결과 코드: `interpolateColors(frame, [0, dur], [startColor, endColor])`
  로 실제 색상 보간 수행 — r3 점수 43 의 직접 원인 (보간 미발생) 해소.

### 비용
- 5-case validation: ~$0.05 OpenAI (gpt-4o-mini 추정).
- TM-46 r4 재측정 예정 비용 ~$0.6 (이전 r3 와 동일 추정).
