# TM-44 — Retro

## What went well

- **Code-path proof beat UI proof.** The Jest+RTL test exercising
  `ParameterControl` directly + asserting on the evaluator cache produced
  stronger, deterministic, reusable evidence than a 10-template Playwright
  sweep would have. 245-test suite (was 230) now codifies the round-trip
  invariant.
- **Two real bugs caught.** The pre-existing `parseFloat(x) || 0` and free-form
  hex input were silent data-quality bugs that would have continued to
  poison `paramValues` for AI-generated assets. Both fixed in this PR with
  surgical, well-commented changes.
- **Architecture validated.** The `useMemo([asset?.jsCode])` + LRU hash cache
  combo means PARAMS edits literally cannot trigger an evaluator rebuild.
  This is now a pinned invariant via test (cache size stays at 1 across 100
  same-jsCode evaluations).

## What hurt

- **Playwright MCP is a singleton.** The plugin spawns a single browser
  shared across all parallel TeamLead sessions. While this TM-44 worker was
  navigating to `localhost:3044/studio`, sibling agents on TM-43/45/46/48
  redirected the same browser tab to their respective dev ports (3043,
  3045, 3048). Each context-destroyed evaluation lost state. Captured one
  clean screenshot before the sweep became infeasible.
- **CSP `media-src` blocks Remotion silent-audio data URI.** 195 console
  errors per session is noise that will mask future real errors. Spawned
  `AI-BUG-csp-media-data` (triggers_requalify: TM-44).
- **NEXTAUTH_URL hardcoding.** `.env.local` is shared across all worktree
  dev processes; the auth callback redirected to whatever port `NEXTAUTH_URL`
  is set to (likely 3000). Worked around by passing per-process
  `NEXTAUTH_URL=http://localhost:3044` + `AUTH_TRUST_HOST=true`. Should be
  documented in `tech-notes` for future QA workers.

## Process improvements (suggested)

1. **Serialize Playwright-dependent tasks** in Orchestrator scheduling.
   TM-44/45/46/47/48 all need MCP Playwright; running them in parallel
   guarantees collisions. A `requires: playwright-mcp` lock would prevent
   this.
2. **Per-worktree env overrides.** Add a `worktrees/TM-XX/.env.local.override`
   convention for `NEXTAUTH_URL` / `AUTH_URL` / `PORT` so each TeamLead's
   dev server is fully isolated.
3. **Code-path QA as default.** When a QA task is fundamentally about state
   propagation (slider→player), prefer Jest+RTL over Playwright. UI
   screenshots are nice-to-have, not load-bearing.

## Decisions

- **No new ADR needed.** ADR-0002 (PARAMS auto-extract) already covers the
  contract. This PR strengthens the boundary but does not change it.
