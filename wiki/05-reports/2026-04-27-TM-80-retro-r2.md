## 잘된 것

1. **fix → requalify 사이클의 결정성** — TM-90 PR이 정확히 r1에서 지목한 룰 3종(`label`, `button-name`, `aria-input-field-name`)을 노드 단위까지 0으로 끝냄. 회귀 검증 비용 = 스크립트 1회 재실행으로 끝.
2. **스크립트 재사용** — r1 작성한 `tm-80-a11y-audit.mjs`가 그대로 r2에서 동작. 코드 변경 없음 → r1↔r2 결과의 직접 비교 가능성 보장.
3. **고립된 워크트리** — 3080 dev + 새 SQLite로 main/타 worktree와 충돌 없이 실행.

## 아쉬운 점 / 마찰

1. **node_modules 부트스트랩 누락** — 새 worktree에 `npm install`을 별도로 돌려야 했음 (`playwright`, `axe-core`, `jest-axe` 미존재). `setup-worktree.sh`가 deps 설치는 안 함(설계상 의도). r2 audit처럼 devDep을 직접 호출하는 task에 한해, 컨텍스트 파일에 "필요 시 npm install" 가이드가 명시되었으면 더 빨랐음.
2. **stale main lockfile 경고** — Next.js Turbopack가 worktree와 main 양쪽의 lockfile을 감지해 경고. 기능 영향은 없으나 추후 `turbopack.root` 설정 권장.
3. **incomplete 카운트 증가** — r1 `incomplete=3` (5 templates) → r2 `incomplete=4`. axe가 contrast 판정 실패 케이스가 1건씩 더 검출됨(아마 violet-600/kbd 케이스의 색샘플링 변동). 보고서에 incomplete 별도 트래킹은 미반영 — 다음 회차에서 룰 ID 단위 분석 룰 추가 권고.
