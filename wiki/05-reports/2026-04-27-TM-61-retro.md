# TM-61 — 회고

## 무엇을 했나

`fluid-blobs` 템플릿의 metaball 합성이 시작 시점에 보이지 않는 시각 버그를 진단·수정. SVG goo 필터의 종단 처리를 표준 패턴(`feComposite operator="atop"`)으로 정정하고, blur stdDeviation 하한을 blob 크기에 비례하도록 보강.

## 잘된 점

- TM-43 audit 보고서가 매우 정확하게 증상을 기록했음 — "start 시점 미합성, mid 에서 합성됨" 이 곧 `feBlend normal` 의 정확한 표현. 즉 audit → fix flow 가 의도대로 동작.
- 임시 dev fixture (`/dev-fluid-preview`) 로 멀티-worktree cookie origin 충돌(TM-43, TM-65) 을 우회 → 동시 작업 환경에서 단일 컴포넌트 시각 검증 패턴 확립.
- 한 번의 코드 수정 + Playwright 캡처 1회로 검증 완료.

## 어려웠던 점 / 발견

- macOS 에서 `127.0.0.X` loopback alias 를 추가하려 했으나 sudo 권한 차단 — 멀티-worktree dev 분리는 OS 단계 대신 별도 fixture 페이지로 우회.
- `feBlend` 의 default mode 가 `normal` 이라는 점은 SVG spec 상으로는 자연스럽지만, "goo recipe" 의 카피-페이스트 출처에 따라 잘못 들어오기 쉬움 — 다른 템플릿에서도 grep 필요.

## 학습 (재발 방지)

- SVG 필터 파이프라인은 마지막 노드가 `result` 를 출력하는지가 핵심. `feBlend` 를 종단으로 쓸 거면 `mode="screen"` 등 명시 — `normal` 은 SourceGraphic 을 그대로 위에 얹는 것과 동일하므로 metaball 효과를 죽임.
- PARAMS 슬라이더가 시각 효과의 핵심 파라미터를 0 근처로 깎을 수 있다면, 컴포넌트 내부에서 effective lower bound 를 보장해야 일관된 시각 품질 유지.
