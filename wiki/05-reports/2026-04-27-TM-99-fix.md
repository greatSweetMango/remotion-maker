# TM-99 — PlayerPanel 재생속도 컨트롤 추가

## TL;DR

TM-79 r1 QA에서 발견된 F1(재생속도 UI 부재)을 해결. `PlayerPanel`이 Remotion `<Player>`에 `playbackRate` + `showPlaybackRateControl`을 전달하도록 수정하고, 툴바에 0.5×/1×/1.5×/2× 버튼을 추가해 사용자가 미리보기 속도를 조절할 수 있게 함. 단위 테스트 5건 통과.

## 무엇이 바뀌었나

- `src/components/studio/PlayerPanel.tsx`
  - `playbackRate` 로컬 state 추가 (default `1`).
  - 상수 `PLAYBACK_RATES = [0.5, 1, 1.5, 2]` 도입.
  - Player 호출에 `playbackRate={playbackRate}` + `showPlaybackRateControl={PLAYBACK_RATES}` 전달.
  - 툴바에 `role="group"` `aria-label="Playback speed"` 컨테이너 + 4개 토글 버튼(`aria-pressed` 동기화) 추가. asset 부재 시 렌더 X.
- `__tests__/components/player-playback-rate.test.tsx` 신규
  - Player를 mock해 props 전달을 검증 (`playbackRate=1`, `showPlaybackRateControl=[0.5,1,1.5,2]`).
  - 버튼 클릭 → state → Player prop 라운드트립.
  - asset 없을 때 control 그룹 미렌더 가드.

## 왜 / 배경

TM-79 r1 QA가 PlayerPanel의 재생속도 props 누락을 코드 감사로 결정적으로 식별했음 (`wiki/05-reports/2026-04-27-TM-79-qa.md` F1). 사용자가 긴 애니메이션을 미리볼 때 1×만 가능해 작업 효율이 떨어졌음. `triggers_requalify: [79]` 박힘 → 머지 시 TM-79 자동 재검증.

## 영향

- **코드/시스템**: 변경은 PlayerPanel 단일 파일 + 테스트 1개. PARAMS / Edit≠Render 등 ADR 영향 없음. 추가 의존성 없음 (Remotion v4 Player 기본 prop 활용).
- **사용자/제품**: Studio 미리보기에 즉시 가시적인 4단 속도 토글이 노출됨. 키보드 a11y는 native button + aria-pressed로 보장.
- **비용/성능**: `playbackRate`는 Remotion 내부 RAF 타임 스케일링만 변경 — 추가 렌더 비용 없음. 자동 다운샘플 latch 로직과 독립.
