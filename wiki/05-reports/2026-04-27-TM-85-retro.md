# TM-85 Retro

## 잘 된 것

- **TM-48 토대 위에서 incremental** — 동일 파일(`sandbox.ts`) 1곳만 건드려서
  blast radius 극소. 기존 80개 테스트 모두 그대로 통과.
- **0-day 2종을 정적으로 잡음** — setTimeout 재귀는 timer 자체 deny로 surface
  봉쇄, Promise self-chain은 휴리스틱 1개로 30 케이스 전부 통과 + benign
  `Promise.then(handler)`은 정상 통과 (false-positive 베이스라인 통과).
- **사용자 메시지 contract 자동 검증** — fuzz 케이스마다 라벨 길이/스택 부재를
  assert해서 향후 라벨 회귀 가드.
- **OpenAI 호출 0** — 단위검증 외 LLM 의존 없음.

## 아쉬운 것 / 발견한 한계

1. **Mutual recursion 미감지** — `f→g→f` 형태의 Promise 상호재귀는 휴리스틱이
   잡지 못함. 실 LLM output에서 미관측이라 우선순위 낮춤. AST 전환 시 자연
   해결.
2. **`global` 토큰 word-boundary 미세** — `useGlobalsOfMine` 같은 식별자는
   benign baseline에서 통과 확인했으나, `global` 단독 사용은 non-Node 환경에서
   존재하지 않음. 실용적으로는 무해, 다만 deny 라벨이 사용자에게 혼란스러울
   여지.
3. **Path 표기 — TM-48 회고에서 한 번 박아둔 함정 재발** — task spec엔
   `src/lib/evaluator/`라 적혀 있었으나 실제 경로는 `src/lib/remotion/sandbox.ts`.
   테스트 폴더는 spec대로 `__tests__/lib/evaluator/`에 만듦 (sandbox-fuzz는
   evaluator deny-list 검증이라는 점에서 적절).

## 다음 회차

- AST 기반 deny 평가 (research task 후보) — 현 휴리스틱이 90% 커버하지만
  graph-based 재귀 검출은 AST 없이는 한계. sucrase output 위 cheap traverse
  비용 측정 필요.
- evaluator → sandbox 통합 에러 채널 — 현재 sandbox `errors[]` 라벨과
  evaluator `userMessage` 한국어 카피가 이원화돼 있음. 토스트 UX 통일 후보.
