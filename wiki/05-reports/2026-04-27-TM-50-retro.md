# TM-50 retro

## What went well

- QA(TM-41/TM-43) 에서 박힌 정확한 stack trace + 발생 횟수(175회+) 로 원인 즉시 특정 → 코드 1라인 + 테스트 1파일.
- `img-src`/`font-src` 가 이미 `data:` 를 허용하던 선례가 있어 정책 결정 부담 0.
- Header 단위테스트가 dev 서버 없이 next.config 의 `headers()` 를 직접 호출해 검증 가능 — 빠른 회귀 가드.

## What could be better

- TM-34 sandbox CSP 도입 시 Remotion 의 audio probe 동작을 미리 검증했다면 TM-41/TM-43 가 깨끗한 콘솔로 측정 가능했을 것. 콘솔-noise 가 QA 신뢰도를 떨어뜨렸음.
- ADR-PENDING-TM-34 본문에 “media-src 의 data: 결정 로그” 가 없어 왜 빠졌는지 추적 불가 — 향후 ADR 은 *exclude 한 source* 도 명시하는 게 좋겠다.
