# TM-60 — type-explosion 재조립(reassembly) 단계 가시성 수정

## TL;DR

TM-43 시각 검수에서 발견된 type-explosion 템플릿의 "분해만 보이고 재조립 X" 이슈를, phase 타이밍 재배치 + reassemble spring 튜닝으로 해결. 6초 영상의 끝 1.5초가 명확한 "EXPLODE" 헤드라인으로 정착하도록 재조립 motion이 화면에 보임.

## 무엇이 바뀌었나

`src/remotion/templates/TypeExplosion.tsx`:

1. **Phase 타이밍 재배치**
   - 이전: explode 0.35→explode+40f, reassemble 0.65부터 (= durationInFrames 360 기준 234~270f, 끝까지 90f 정착)
   - 이후: explode 0.30→0.50, reassemble 0.55부터 시작, 0.80에서 거의 1 도달, 0.80~1.0에 헤드라인 정착
2. **Reassemble spring 튠 다운**
   - 이전: `damping:14, stiffness:110, mass:1, stagger:i*1.2` → ~20프레임 만에 1로 점프 (시각적으로 jump cut)
   - 이후: `damping:22, stiffness:60, mass:1.2, stagger:i*2` → ~75프레임(1.25s) 동안 부드럽게 회귀, 글자별 stagger도 더 큼
3. **Explosion distance 축소**
   - 이전: `220 + rnd*480 px` (force=1 기준 최대 700px) → 1920×1080 캔버스 가장자리 근처
   - 이후: `180 + rnd*360 px` (최대 540px) → 화면 안에 머무르며 분해 형태가 시각적으로 잘 읽힘
4. **코멘트(phase plan)도 실제와 일치하도록 갱신**

## 왜 / 배경

- 출처: [[2026-04-27-TM-43-template-visual-audit]] 의 "시각 부적절 4건" 중 type-explosion (#1).
- 코드 자체엔 reassemble 로직이 존재했으나, durationInFrames=360에서 reassembleStart=234. 즉 6초 중 끝 1.5초에 회귀가 일어나야 했고, 실제로는 spring 파라미터가 너무 stiff해서 ~20프레임(0.33s) 만에 home으로 jump → 시각적으로 "흩뿌려진 채로 끝났다"고 인지됨.
- TM-43 캡처 타이밍(start/mid/end)이 모두 explode 직전~explode 진행 중에 몰려 있어 audit에서 더 두드러져 보였음(end 캡처도 frame≈116, 즉 explode 시작점). 그러나 본질은 "재조립이 시각적으로 보이지 않는 spring tune"이었음. 새 튠으로는 explode peak가 frame 180(3s) 부근, reassemble 진행이 frame 200~270(3.3s~4.5s)에 완전히 화면 안에서 진행되며, 4.5s 이후 1.5s 동안 헤드라인이 머문다.
