# TM-48 Retro

## 잘 된 것

- **단위 검증 비용이 거의 0** — 31 jest 테스트 1초 미만, OpenAI 호출 0.
- **Backward-compat 유지** — `evaluateComponent()` 시그니처 그대로 둬서 5개 호출처(TemplateCard/TemplatePicker/PlayerPanel/SharePlayer/UniversalComposition) 중 PlayerPanel 1곳만 수정. 회귀 위험 최소화.
- **Sandbox layer로 끌어올린 무한 루프 차단** — evaluator의 wall-clock 타임아웃이 사후 측정이라 sync 무한 루프를 못 끊는 한계를 인정하고, 사전 정적 deny로 보완 (`for(;;)`, `while(true)`).
- **dev-only fixture 페이지** — Playwright가 OpenAI/auth 흐름을 거치지 않고 evaluator+UI를 직접 검증하게 해줘서 통합 테스트 사이클이 짧음.

## 아쉬운 것 / 발견한 한계

1. **Remotion Player의 자체 fallback이 우리 ErrorBoundary를 가로챔** — `() => { throw new Error('boom'); }` 케이스에서 Player가 ⚠️ 한 글자만 표시. 우리 친절 메시지 미적용. 별도 fix task 후보.
2. **첫 줄 코멘트 후 동일 줄 component 선언이 regex에 안 잡힘** — `(?:^|\n)\s*` 가 줄바꿈 또는 문서 시작만 매칭. `/* x */ const Comp = () => null;` 1줄 입력은 missing-component. fuzz #17은 newline 추가로 우회. LLM 출력은 보통 줄 분리되므로 영향 작음, 그러나 알아둘 함정.
3. **Path 표기 혼란** — task spec에 `src/lib/evaluator/`로 적혀 있었으나 실제 경로는 `src/lib/remotion/evaluator.ts`. context 파일에 명확히 박아둠.
