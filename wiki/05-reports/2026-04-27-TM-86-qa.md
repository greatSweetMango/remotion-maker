## TM-86 spec vs 실제 구현 — 불일치 보고

본 task spec 은 다음 항목을 요구:
- "drag-reorder, scene 분리, scene 추가/삭제, sidebar resize"

하지만 ADR-0008 / TM-28 / 현재 코드의 sequence-aware sidebar 는 **read-only filter + navigation** 만 제공 (수정·재배치·추가·삭제 X):
- Sequence 정보는 **소스 코드의 `<Sequence>` JSX 정규식 파싱** 결과 — 사용자 편집 대상이 아님
- 셀은 click-to-seek + sticky-select 만 가능 (드래그 핸들 없음)
- scene CRUD 는 prompt → LLM 재생성 경로로만 가능 (이는 TM-28 범위 밖이며 본 ADR 의 명시적 비-목표)

따라서 spec 문구의 "drag-reorder / 분리 / 추가 / 삭제"는 **검증 불가** (기능 부재).
정확한 검증 가능 항목(filter / sticky / All / hotkey / resize)은 모두 PASS 처리.

후속 ADR 로 "Editable scene structure (TM-?)" 가 필요하다면 별도 task 로 분리할 것.

## Gotcha 발견

Playwright MCP 브라우저는 다른 worktree agent(예: TM-79 on :3079)와 **공유**되어 다른 agent 의 navigation 으로 탭이 전환됨 — `browser_tabs` 로 명시 select 필요. 또 NextAuth dev cookie 가 host(127.0.0.1)만 일치하면 port 무관하게 공유되어 cross-worktree session leakage 가능.

→ tech-note 후속 작성 권장: `2026-04-27-playwright-mcp-shared-browser.md` (TM-65 다음 단계). 본 보고서 시점에는 회피책(`browser_tabs select 0` + `history.replaceState`) 만으로 진행.
