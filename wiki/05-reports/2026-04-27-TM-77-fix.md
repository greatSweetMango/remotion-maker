# TM-77 — spend.json 자동 갱신 누락 수정

## TL;DR

ADR-0006의 PostToolUse hook은 Claude Code 자체 도구 호출만 잡고, 우리 앱의
Anthropic/OpenAI 직접 호출은 잡지 못해 `current.cost_usd=0`, `openai_total_usd=0`
이 고착되어 있었다 (TM-49 발견). `src/lib/ai/spend.ts` 모듈을 추가하고
`chatCompleteStream`에서 양 provider 응답의 usage를 즉시 적재. hook도
OpenAI shape (`prompt_tokens`/`completion_tokens`)를 인식하도록 보강.

## 무엇이 바뀌었나

- 신규: `src/lib/ai/spend.ts` — `recordUsage()`, 가격표 (Anthropic 4.5 / OpenAI gpt-4o*),
  flock-equivalent 단일 파일 갱신, `SPEND_FILE_PATH` env 오버라이드 지원.
- 수정: `src/lib/ai/client.ts`
  - OpenAI 스트리밍에 `stream_options.include_usage=true` 옵트인 → 마지막 청크의
    `event.usage`를 `recordUsage({provider:'openai', ...})`로 적재.
  - Anthropic 스트리밍 종료 후 `stream.finalMessage().usage`를 읽어
    `recordUsage({provider:'anthropic', ...})` 적재 (cache_creation/cache_read 포함).
- 수정: `.claude/hooks/post-tool-use.sh`
  - 모델 family 분기 → provider 분기 (`gpt-*` / `*o1*` / `*openai*` → openai)
  - usage 추출에 OpenAI 키 (`prompt_tokens`/`completion_tokens`) fallback 추가
  - OpenAI 분기에서 `openai_total_usd`도 함께 갱신
- 신규 테스트: `__tests__/lib/spend.test.ts` (7 통과), `__tests__/lib/spend-hook.test.ts`
  (3 — jq 미설치 환경 skip).

## 왜 / 배경

- TM-49에서 `current.cost_usd`가 0에 박혀 있고 야간 자율 예산 캡이 트리거되지
  않는다는 증거 발견.
- ADR-0006이 가정한 PostToolUse 페이로드의 `tool_response.usage`는 Claude Code
  세션이 호출하는 sub-agent Task 도구에서만 채워진다. 우리 앱의
  `chatCompleteStream`은 Claude Code의 도구가 아니라 Next.js 서버 코드여서
  hook 자체가 발화하지 않는다.
- 따라서 ADR-0006의 약속을 지키려면 (1) 앱 코드에서 직접 spend.json에 적재하거나
  (2) hook 외 메커니즘이 필요. (1)을 채택해 hook과 같은 파일·같은 락을 공유.

## 영향

- 코드: `src/lib/ai/client.ts` 양쪽 provider 경로에 try-catch로 감싼 sink 추가.
  실패해도 응답 스트림에는 영향 없음 (cost ledger never blocks request path).
- 운영: 야간 PM 루프 (`session-start.sh`의 95% 게이트)가 실제 누적치 기반으로
  STOP을 걸 수 있게 됨. `openai_total_usd`도 자동 갱신 → ADR-0019의 OpenAI QA 캡
  ($18) 이 의미를 가진다.
- 비용: 토큰 측정 자체에는 추가 비용 없음 (이미 SDK가 usage를 반환).
