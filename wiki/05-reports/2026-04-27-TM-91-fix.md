# TM-91 — Customize panel undo/redo

## TL;DR

`useStudio` 리듀서에 `{past, future}` 히스토리 스택을 추가해 customize 패널 파라미터 편집을 undo/redo 가능하게 했다. Cmd+Z / Cmd+Shift+Z 글로벌 단축키 + UI 버튼(↩ ↪) 노출. depth=100 cap, 분기 시 future clear, SET_ASSET / ADD_VERSION / RESTORE_VERSION 시 히스토리 리셋.

## 무엇이 바뀌었나

- `src/types/index.ts`: `StudioState`에 `history: { past, future }` 추가, `UNDO`/`REDO` 액션 타입 추가
- `src/hooks/useStudio.ts`:
  - 리듀서를 export, `initialState`/`HISTORY_DEPTH` 상수 export (테스트용)
  - `UPDATE_PARAM` — 같은 값 쓰기는 no-op, 다르면 이전 paramValues를 past에 push, future clear, depth>100이면 가장 오래된 것 shift
  - `UNDO` — past pop → present, present → future unshift
  - `REDO` — future shift → present, present → past push
  - `SET_ASSET` / `ADD_VERSION` / `RESTORE_VERSION` 시 history 리셋 (파라미터 셋이 달라지므로 cross-asset undo 의미 없음)
  - 글로벌 keydown 리스너: Cmd/Ctrl+Z, Cmd/Ctrl+Shift+Z, Cmd/Ctrl+Y. INPUT/TEXTAREA/contentEditable 포커스 시 패스스루 (브라우저 텍스트 undo와 충돌 회피)
  - hook return에 `undo, redo, canUndo, canRedo` 추가
- `src/components/studio/CustomizePanel.tsx`: 헤더에 `Undo2`/`Redo2` 버튼, disabled 상태 바인딩, aria-label 포함
- `src/components/studio/Studio.tsx`: desktop + mobile 모두 props wiring
- `__tests__/hooks/studio-history.test.ts` (신규, 8 cases): 50회 변경/undo, 분기, depth cap, no-op, redo, asset 전환 시 reset

## 왜 / 배경

TM-78 검수에서 customize 패널이 한 번 슬라이더를 옮기면 되돌릴 수 없다는 점이 발견됐다 (versions 시스템은 prompt edit 단위라 파라미터 미세조정에는 too coarse). UX 표준상 미세 편집은 undo가 기본 기대치.

## 영향

- 코드: 상태 모양 확장 (history 필드). 기존 reducer cases는 history 보존하므로 호환. 컴포넌트 props는 옵셔널이라 기존 테스트 영향 없음 (customize-roundtrip 15/15 통과 확인).
- 사용자: 파라미터 100단계까지 undo 가능. 키보드 단축키 지원으로 디자이너 워크플로우 개선.
- 메모리: 스냅샷 1개 ≈ 파라미터 수 × (key + primitive). 일반 템플릿 ~30 params → 100 deep = 수백 KB 수준. 무시 가능.

## 결정 / 트레이드오프

- 히스토리에 paramValues 객체 자체를 저장 (vs diff). 이유: 단순/디버깅 용이/메모리 비용 무시 가능.
- 같은 값 UPDATE_PARAM은 history 푸시 X — 컨트롤드 인풋 리렌더로 인한 스팸 방지.
- INPUT/TEXTAREA 포커스 시 단축키 비활성. 텍스트 input 안에서 Cmd+Z는 브라우저 텍스트 undo가 처리하도록 양보. ContentEditable 동일.
- Asset 전환/버전 추가/복원 시 히스토리 리셋: 다른 파라미터 셋의 스냅샷은 의미 없음 (없는 키 부활 위험).
