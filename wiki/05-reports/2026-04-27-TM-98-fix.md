# TM-98 — Studio responsive layout fix (mobile + iPad)

## Summary
TM-84 QA flagged that the Studio root layout broke at sub-`md` viewports:
- iPhone 375 — Player rendered 0×0 and the Customize tab was unresponsive.
- iPad portrait 768 — right-side Customize panel sat outside the viewport
  and the top toolbar was partially hidden.

Both stem from a single root layout flaw (3-pane `PanelGroup` gated on the
default Tailwind `md:` breakpoint, which is **768px**). At exactly 768
the desktop layout activates with no slack: `22% + 52% + 26% = 100%` of
viewport, but inner toolbars (Player FPS toggle + bg swatches, header
title + share + dashboard link) push past the panel widths and get
clipped. On phones the previous mobile layout only mounted `<PlayerPanel>`
under the *Customize* tab, so the first paint always had Player at 0×0.

## Resolution
1. Bumped the desktop/mobile gate from `md:` (≥768) to `lg:` (≥1024).
   iPad-portrait now uses the responsive stacked layout.
2. Restructured the mobile/tablet stack to mount **`<PlayerPanel>`
   persistently** at the top (40-45vh), with the bottom region swapping
   between Prompt / Customize / Export. TemplatePicker takes over when
   no asset is loaded, fixing the previous "blank screen on first load"
   path.
3. Header — added `flex-wrap`-friendly classes plus `min-w-0` /
   `flex-shrink-0` so the brand block stays put while the right cluster
   condenses. Hid the asset title and Dashboard link below `lg`, abbreviated
   the Upgrade CTA below `sm`.
4. PlayerPanel toolbar — `flex-wrap` + `min-w-0` on the title; hid the bg
   swatch row below `sm` (still accessible on tablet and up).
5. Side-fix: `src/lib/usage.ts` exported `TIER_LIMITS` from a module that
   imports prisma. PromptPanel (a client component) imports `TIER_LIMITS`,
   so the chunker dragged `node:fs` into the browser bundle and Next 16
   Turbopack failed with *"chunking context (unknown) does not support
   external modules"*. Split the constant out to a leaf
   `src/lib/tier-limits.ts` and re-exported from `usage.ts` for back-compat;
   PromptPanel now imports from the leaf. This unblocked the dev server
   for verification and removes the latent client-bundle regression.

## Retrospective

### What went well
- Single-file diagnosis at the root (`md:` boundary) avoided per-template
  patching that the spec explicitly warned against.
- Persistently mounted Player solves the "0×0 on first paint" symptom
  without resorting to ResizeObserver hacks.
- Discovered + fixed a latent Turbopack client-bundle issue while setting
  up dev verification.

### What could be better
- Verification was blocked for ~10 min by the pre-existing client-bundle
  prisma issue; a tech-note or status flag for known turbopack chunking
  bugs would have shortened that.
- Tailwind `md:` (768) being inclusive of iPad-portrait is a recurring
  trap; could codify a project-wide `lg:`-as-desktop convention in
  `wiki/02-dev/tech-notes/`.

### Action items
- Consider a tech-note on the prisma client-bundle pitfall (separate task).
- No new ADR — change uses existing breakpoint conventions.
