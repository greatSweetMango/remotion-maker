# TM-47 회고

## 결과

- 코드 변경: `src/app/api/generate/route.ts` 에 `[generate] tier=… model=… usage=…` 진단 로그 1줄 추가
- 검증 산출물: `wiki/05-reports/2026-04-27-TM-47-qa.md`
- 5×3 = 15개 호출 (비로그인 5, FREE 5, PRO 5) 모두 acceptance 부합
- 신규 버그 0, spawned task 0

## 잘된 점

- API 라우트가 라우트 핸들러 단계에서 직접 `auth()` 호출 → middleware matcher 차이에도 401 안정적
- FREE quota 차단 메시지가 `Upgrade to Pro for 200/month.` 로 사용자 행동 유도 명확
- clarify 응답이 quota 미차감 — 모호 프롬프트로 quota 고갈 어뷰즈 차단 (route.ts:46-49). 정책으로 명문화 가치 있음

## 고민

- proxy.ts matcher 가 `/api/*` 를 제외 → 신규 API 라우트 추가 시 `auth()` 호출 누락하면 그대로 미인증 통과. 컨벤션 강제 수단 (lint rule, helper wrapper) 부재. 본 task 외 follow-up 후보.
- Next.js Turbopack 의 root 추론으로 worktree 격리가 흐려짐 — `prisma/dev.db` 가 부모 디렉터리 것 사용됨. 부모 DB 백업 후 SQL 조작 → 테스트 끝나도 backup 복구 안 함 (PR 머지 후 orchestrator cleanup 시 처리).
- console.log 한 줄을 영구 코드로 남기는 게 적절한지 — 프로덕션 로그 레벨 제어 부재. 추후 `pino` 같은 logger 도입 후 통합 가능.

- (옵션) clarify-no-quota 정책을 ADR-0004 로 명문화
