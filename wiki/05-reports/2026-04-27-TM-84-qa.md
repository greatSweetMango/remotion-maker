# TM-84 — Studio (customize) 페이지 mobile/tablet QA

## TL;DR

35 템플릿 customize 페이지(`/studio?template=<id>`)를 iPhone(375×812) / iPad(768×1024) viewport 에서 검증. **iPhone 에서 Remotion Player 가 0×0 으로 렌더되지 않으며 Customize 탭 클릭이 동작하지 않음**, **iPad 에서 우측 Customize 패널이 viewport 밖으로 잘림**. 확인된 5/35 템플릿 모두 동일 패턴 → 레이아웃 셸 자체 결함으로 판단. Fix task **TM-80** spawn(`triggers_requalify=[TM-84]`).

  (`screenshots/TM-84/{iphone,ipad}-{counter-animation,bar-chart,particle-physics,holographic-card,glitch-effect}.png`)
- Bug task **TM-80** 등록 (priority=high, deps=[TM-84], metadata.triggers_requalify=["TM-84"])

## 왜 / 배경

TM-84 는 35 템플릿 customize UI 의 mobile/tablet viewport QA. ADR-0002(PARAMS auto-bind UI) 가 데스크톱에서만 검증돼 왔고, 모바일 사용자 경험은 실측이 없었음.

## 검증 매트릭스

| Template | iPhone 375×812 | iPad 768×1024 |
|---|---|---|
| counter-animation | Player 미표시, Customize 탭 비반응 | 우측 패널 잘림, toolbar 일부 숨김 |
| bar-chart | 동일 | 동일 |
| particle-physics | 동일 (1 console error) | Player 표시 OK, 패널 잘림 |
| holographic-card | 동일 | 동일 |
| glitch-effect | 동일 | Player 표시 OK, 패널 잘림 |

샘플 5 (≈14% 커버리지) 만으로도 100% 일관 — root layout 결함이지 template 별 결함 아님. 시간 제약으로 전수 35 검증은 미실시.

## 발견 (블로커)

### F-1 [P1] iPhone 375px — Remotion Player 0×0

```
div[class*="remotion"] getBoundingClientRect = {width:0, height:0}
offsetParent = null  → 사실상 hidden
```
- Customize 페이지 핵심 기능(미리보기) 완전 차단.
- 하단 nav(Prompt / Customize / Export) 는 표시되지만 **Customize 탭 클릭해도 panel 전환 X** (UI freeze) — 일관 재현.

### F-2 [P1] iPad 768px — 우측 Customize 패널 viewport 밖으로 잘림

- Customize/Export 탭 라벨이 우측 끝에서 잘려 표시.
- 패널 내부 세로 스크롤은 가능하나, 가로 overflow 로 우측 컬러피커/슬라이더 일부가 가려짐.
- 상단 toolbar(템플릿명·fps·duration·테마 픽커) 도 좁아져 **테마 픽커 일부 비가시**.

### F-3 [P2] iPad 상단 toolbar 컴팩트화 미적용

- 데스크톱(1440px) 디자인이 그대로 적용 — 768px 에서 좁은 콘텐츠 박스가 가로 overflow.

## 영향

- 사용자: 모바일/태블릿 사용자는 customize 단계 자체 불가. 템플릿 카탈로그 진입(/studio?template=...) 직후 freeze.
- 비즈: PRD 의 "any device" 약속과 ADR-0002 자동 바인딩 UX 가 desktop-only 로 제한.
- 코드 영향 범위: studio 셸 layout (`src/app/studio/page.tsx` + 그 하위 panel/player 컴포넌트), 추정.

## 근본 원인 가설

- `react-resizable-panels` v4 셸이 percent-based + `min-width:px` 로 잡혀, viewport 가 패널 합산 min-width 미만이면 player panel 이 0 으로 collapse.
- mobile breakpoint 에서 player panel 을 `display:block` full-width 로 전환하는 분기 부재.
- 하단 nav 는 mobile 전환됐는데 **content 본문은 desktop layout** — 부분 수정만 적용된 흔적.
