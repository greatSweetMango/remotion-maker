## 잘된 점

- in-app loader 와 동일 파이프라인(sanitize → sucrase → evaluator) 재사용 → 캡처가 실제 유저 화면과 1:1
- dedicated `vr-entry.tsx` 로 export-route 와 결합 분리 — 기존 코드 0 수정
- sharp 활용으로 deps 0 유지하면서 3-way 검증(sha256 / dHash / pixelDiff)
- update / compare 양 모드 단일 driver — `--only=<id>` debug 옵션
- failure 시 `<id>.current.png` 자동 dump → 인간 검수 즉시 가능

## 고민

- baseline PNG 합 ~13 MiB commit. PR diff 가독성에는 영향 미미하지만 첫 commit 한정
- frame=fps(1초) 단일 캡처 — 후반부 애니 회귀 미감지. multi-frame sweep 은 비용/검증력 trade-off
- `UniversalComposition` 이 `registerRoot` 누락된 채로 export-route 에 사용되고 있는 것 발견 — TM-75 본선과 무관해 별도 task 로 escalate 권장
- pHash threshold (5% pixel + Hamming 12bit) 는 첫 CI 결과 보고 튜닝 필요할 수 있음
