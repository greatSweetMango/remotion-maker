## RCA — 진짜 회귀

라이브로 코드 변경 적용은 되는데 사용자가 "동작 X"로 인지한 이유를 추적.

### 버그: 템플릿 백업 asset edit 시 client-side id pivot 누락

```
1) 사용자 → /studio?template=counter-animation 진입
   state.asset.id = "template-counter-animation"

2) 1회차 edit
   client → POST /api/edit { assetId: "template-counter-animation", currentCode }
   server  ← isTemplate true → prisma.asset.create() → savedAssetId = cmA
            response { id: "cmA", code, jsCode, parameters, ... }
   client → ADD_VERSION { code, jsCode, parameters }   ← data.id 무시
   ⇒ state.asset.id 여전히 "template-counter-animation"
   ⇒ URL 여전히 ?template=counter-animation

3) 2회차 edit
   client → POST /api/edit { assetId: "template-counter-animation", currentCode: <1회차 결과> }
   server  ← isTemplate true → prisma.asset.create() **다시** → savedAssetId = cmB
            (cmA은 orphan; monthlyUsage +1 또 차감됨)
   server.editUsage."template-counter-animation" → 2 (path-key 누적)

4) 3회차도 동일 → editUsage."template-counter-animation" = 3
5) 4회차 → reserveEditSlot affected=0 → **429 Edit limit reached (3 per asset on FREE plan)**
   사용자 인지: "edit가 안 된다"
```

표면 증상 4가지 (모두 같은 root cause):
- a. **3회 후 모든 edit이 429**, 실제 DB row는 매번 새로 만들어 졌음에도.
- b. **각 template edit이 monthlyUsage +1 추가 차감** (route.ts L111: `monthlyUsage: { increment: 1 }`). FREE 3 generations만으로 quota 소진.
- c. **새로고침 → URL이 ?template=…로 남아있음 → 원본 템플릿 다시 로드 → 모든 edit 사라짐.**
- d. DB orphan rows: 매 template-edit마다 user.assets에 사용자가 추적 못 하는 row가 생김.

특히 **(c) "edit 결과가 사라진다"** 가 사용자 보고와 가장 자연스럽게 매핑됨.

### Root cause

`src/hooks/useStudio.ts` 의 `edit` callback이 server response의 `data.id` 를 읽지 않음. `ADD_VERSION` action은 `code/jsCode/parameters` 만 업데이트하고 `state.asset.id` 는 건드리지 않음. 따라서 첫 template edit 이후에도 client는 자신을 "template asset"으로 계속 인지.

라우트 자체는 의도대로 동작 (template→DB row 1-shot 머터리얼라이즈). 클라이언트가 그 신호를 무시한 게 회귀.

회귀 도입 시점: 템플릿 picker 도입 + /api/edit template 분기 (`64737f0 fix: handle template-based asset edits`)부터 잠재되어 있던 latent bug. TM-93 atomic UPDATE 머지 후 path-key 누적이 정확해지면서 (이전엔 race로 `<3` 케이스가 더 빈번했을 가능성) 사용자 인지 가능한 수준으로 surface 됨.

## Fix

세 군데 변경:

### 1. `src/types/index.ts` — `ADD_VERSION` action에 optional `newAssetId`/`newTitle`

서버가 fresh DB row를 머터리얼라이즈했음을 client가 표현할 수 있게 action 시그니처 확장.

### 2. `src/hooks/useStudio.ts`

- `edit` callback: `data.id !== state.asset.id` 일 때 `newAssetId` 를 dispatch.
- `ADD_VERSION` reducer: `newAssetId` 가 있으면 `state.asset.id` 도 함께 pivot. 없으면 종전과 동일.

### 3. `src/components/studio/Studio.tsx` — URL 동기화

`state.asset.id` 가 `template-` prefix가 아닌 cuid가 되면 `?template=…` → `?asset=<cuid>` 로 `router.replace`. 새로고침 시 사용자가 자신의 edit 결과로 다시 진입.
