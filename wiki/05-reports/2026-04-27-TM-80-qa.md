# TM-80 — Accessibility Audit (axe-core + Playwright)

## 주요 violation 종류 & fix 권고

### Critical

1. **`label` (Form elements must have labels)**  
   - 위치: `ParameterControl` 내 `<input type="number">` (선택자 예: `input[min="40"]`, `.w-20`).  
   - 원인: 슬라이더 보조 number-input 에 visible `<label>` 또는 `aria-label` 미설정.  
   - **Fix**: `ParameterControl.tsx` 의 number input 에 `aria-label={param.label}` (또는 `htmlFor`-paired `<label className="sr-only">`) 추가.

2. **`button-name` (Buttons must have discernible text)**  
   - 위치: `.peer` 셀렉터 — Switch/Checkbox 토글 버튼.  
   - 원인: shadcn `Switch` 컴포넌트가 wrapping `<label>` 없이 단독 button 으로 렌더되는 경우.  
   - **Fix**: `Switch` 사용 지점에 `aria-label` 또는 인접 `<Label htmlFor>` 추가. `ParameterControl` boolean 분기에서 적용.

### Serious

3. **`aria-input-field-name` (slider 핸들)**  
   - 위치: shadcn `Slider` 의 `<span role="slider" aria-valuemin/max>` 에 `aria-label` 없음.  
   - **Fix**: `<Slider>` wrapper 에 `aria-label={param.label}` 전달, 또는 자식 thumb 에 prop 으로 forward.

4. **`color-contrast` (WCAG AA 4.5:1 미달)**  
   - 발생 빈도: 78 곳 중 70+ 가 customize 사이드바 + studio 칩.  
   - 대표 위반: `text-slate-500` on `bg-slate-900`, `bg-violet-600` 위 흰 `<kbd>` 배경 토큰.  
   - **Fix**: `text-slate-500` → `text-slate-400` 이상으로 톤 업, 또는 배경 명도 조정. `kbd` 토큰은 `bg-slate-700/60 text-slate-100`.

### unlabeled interactive 요소 (kbd `withoutVisibleLabel`)

- studio 페이지에서 5–16개 / 페이지. 대부분 icon-only buttons (HistoryGraph, ResourcePanel toolbar). `aria-label` 보강 필요.

## 키보드 nav 평가

- **Tab 흐름**: 모든 페이지에서 Tab 으로 순차 도달 가능. 트랩(focus loop) 없음. login 은 9 interactive → Tab 11회 통과(Suspense fallback 내 추가 focusable).
- **Esc / Enter**: dialog/menu 가 페이지 초기 상태에 닫혀있어 직접 검증 불가. 후속 task 에서 모달(Export/Share) 열림 상태 audit 필요.
- **focus visible**: outline 적용 여부는 페이지별 표본(8개)으로 sample.outline 기록(results.json 참조).

## 영향

- 코드/시스템: 신규 audit 스크립트 + devDep 추가만. 런타임 코드 변경 없음.
- 사용자: 현재 critical 21, serious 77 → screen reader 사용자에게 customize 패널 항목 식별 어려움. WCAG AA 미준수.
