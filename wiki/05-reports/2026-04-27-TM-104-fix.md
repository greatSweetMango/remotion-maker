# TM-104 — 긴 영상 지원

## TL;DR

TM-102 multi-step pipeline의 `scenes[]` 캡(4)을 12로 확장하고, 사용자 prompt에서 duration hint("60초", "2 minutes", "1m30s" 등)를 추출해 outline 단계에서 N개 시퀀스로 자동 분배. 60s → 4씬, 120s → 8씬, 각 시퀀스 fps/durationInFrames 정확 산출. LLM이 directive를 무시하면 `enforceScenePlan`이 사후 보정.

## 무엇이 바뀌었나

- `src/lib/ai/pipeline.ts`
  - `extractDurationHint(prompt)`: KO/EN/혼합("60초","2분","1분 30초","60s","2 minutes","1m30s") 휴리스틱 추출.
  - `planSceneCount(seconds)`: 5s→1, 30s→2, 60s→4, 120s→8 (≤MAX_SCENES=12).
  - `planSceneDurations(seconds, fps, sceneCount?)`: 정확히 합산되는 frame 분배 (마지막 씬이 반올림 drift 흡수).
  - `enforceScenePlan(outline, plan)`: LLM이 duration directive를 무시한 경우 narrative 유지 + duration 강제 재배분.
  - `generateOutline`: long-form prompt 감지 시 user message에 `DURATION DIRECTIVE` block 부착, 결과가 fps 1초 이상 어긋나면 plan 강제.
  - `MAX_SCENES`를 4 → 12로 상향 (`validateOutline` 캡 메시지도 갱신).
- `src/lib/ai/prompts.ts`
  - `OUTLINE_SYSTEM_PROMPT` 룰 §1: "1–4 scenes" → "1–12 scenes" + 길이별 권장 (≤10s 1–2씬, 10–30s 2–4씬, 30–120s 4–12씬). DURATION DIRECTIVE 따라야 함을 명시.
- `__tests__/lib/ai/pipeline.test.ts`
  - 기존 "caps at 4" → "caps at 12" 갱신 (12씬 OK / 13씬 throw).
  - 신규 11개 케이스: 한국어/영어/혼합 hint 추출, scene plan 합산, short(10s) vs long(120s) outline 동작, directive 사후보정, `enforceScenePlan`.

## 왜 / 배경

ADR-0020 multi-step pipeline은 5초 데모 이내 prompt를 가정해 4씬 캡으로 설계됐다. 60초 마케팅 영상 / 120초 쇼츠 등 long-form 요구가 들어오자:
- 캡 때문에 outline 자체가 throw.
- LLM이 캡을 모르고 5씬 리턴 → throw.
- 캡을 풀어도 LLM이 5초 산출물을 만들어버려 사용자 의도와 불일치.

해결: prompt 단계에서 duration을 결정 → outline LLM에 directive로 강제 → 결과 검증 후 어긋나면 코드로 재정렬. AI 비결정성을 deterministic post-fix로 흡수.

## 영향

- **코드**: pipeline.ts +120 LOC, prompts.ts ±2 LOC, 테스트 +130 LOC. 단일-파일 변경 5개 미만.
- **사용자**: "60초 보라색 마케팅 영상" 같은 자연어 hint가 동작. directive를 안 쓰면 기존 5초 default 유지 (호환).
- **비용**: 12씬 multi-step ≈ 5.6× single-shot 토큰. 기존 `MULTI_STEP_COST_RATIO_WARN=1.7` 가 자동으로 경고 발화. 사용자가 명시적 opt-in 가능.
- **회귀**: TM-102 기존 30개 테스트 모두 PASS (cap 테스트 한 개만 12로 갱신).
