# TM-100 — Placeholder retry budget 2→3 + fallback unblock

## TL;DR

P0 user report: AI returned placeholder twice → user blocked. Extended retry
budget 2→3 with forced-RAG + STRICT reinforcement on the 3rd attempt; on
3-strikes we now substitute a built-in template + warning toast instead of
throwing, so the user is never fully blocked. 19/19 unit tests green.

## 무엇이 바뀌었나

- `src/lib/ai/generate.ts` — `generateAssetPlaceholderRetry` extended from
  one retry to two. New ladder:
  - Attempt #1: standard prompt + RAG (TM-74 unchanged).
  - Retry #2: `GENERATION_NON_EMPTY_REINFORCEMENT` (existing TM-51 prompt).
  - Retry #3: forced-RAG (default counter exemplar if no category matched)
    + new `GENERATION_NON_EMPTY_REINFORCEMENT_STRICT`.
  - 3-strikes → `buildFallbackAsset` returns a CounterAnimation-based
    asset with `warning` field (no exception).
- `src/lib/ai/prompts.ts` — added `GENERATION_NON_EMPTY_REINFORCEMENT_STRICT`:
  explicit "≥200 chars + `const PARAMS = {` + `<AbsoluteFill>` root +
  `useCurrentFrame`/`interpolate`/`spring`" requirements; forbids
  `mode="clarify"` on this attempt; commands sensible defaults.
- `src/lib/ai/retrieval.ts` — added `retrieveForcedReferenceForPrompt` that
  always returns an addendum (uses `CounterAnimation.tsx` as default
  exemplar when category inference returns null).
- `src/types/index.ts` — `GenerateApiResponse.generate` gets optional
  `warning?: string`.
- `src/app/api/generate/route.ts` — passes `warning` through to client.
- `src/hooks/useStudio.ts` — surfaces `warning` as 8s `toast.warning`
  ("default template substituted, please refine prompt").

## 왜 / 배경

User reported P0 after merging TM-51 / TM-67 / TM-71 / TM-74 — gpt-4o-mini
still returned 25-char `() => null` stubs twice in a row, throwing the
"placeholder twice" error and blocking the user.

Hypotheses (from user brief):
1. FREE (gpt-4o-mini) too weak for complex prompts → confirmed: small
   models hit placeholder pattern more often.
2. Retry reinforcement not strong enough → addressed by STRICT prompt.
3. RAG addendum may not fire on short prompts → addressed by forced
   default reference on 3rd retry.
4. 2 retries insufficient → upgraded to 3.

Instead of just adding retries, we also added a **fallback**: if even the
3rd attempt fails, return a working built-in template with a `warning` so
the user can edit something concrete rather than face a hard error.

## 영향

- **코드** — generate.ts +95/-20, prompts.ts +43/+0, retrieval.ts +75/+0,
  types +1/-1, route +6/-1, useStudio +9/-2.
- **사용자** — P0 unblocking. Worst case is now a generic counter template
  + warning toast, never a hard "rephrase your prompt" error wall.
- **비용** — worst-case +1 OpenAI call per generation (3 instead of 2).
  Estimated +50% cost on the placeholder-retry path only. Most prompts
  succeed on attempt #1 so amortized cost increase is negligible (<5%).
- **테스트** — generate.test.ts: 19/19 (was 18; +2 new TM-100 tests, -1
  obsolete "throws on 2 strikes" replaced).
