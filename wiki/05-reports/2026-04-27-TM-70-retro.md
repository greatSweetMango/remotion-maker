# TM-70 회고 — TM-46 r3→r4 회귀 RCA

## TL;DR

- 가설 (1) judge variance 가 dominant 임을 30 OpenAI 호출 (5 sample × 2 condition × 3 repeat) 로 확정.
- aggregate per-sample Δmax = **10.4** (default temperature=1.0) → **1.4** (temperature=0, seed=42).
- 즉시 조치: `tm-46-judge.ts` 에 `temperature: 0, seed: 42` 추가.
- 박제: RCA 보고서 + ADR-0018 (N=3 acceptance gate 정책).
- triggers_requalify=[46] 로 머지 시 TM-46 자동 r5 회차 전이.

## 잘 된 점

1. **재현 비용 0 으로 가설 검증** — TM-43 박제 PNG 셋을 fixture 로 재사용해 PNG 재캡처 없이
   judge-only variance 를 분리. dev 서버 불필요, OpenAI ~$1.2 만 소요.
2. **A/B 동일 코드 경로** — variance 실험이 `tm-46-judge.ts` 의 system prompt/스키마/모델/이미지
   파이프라인을 그대로 호출하도록 만들어, 측정값을 production judge 에 1:1 적용 가능.
3. **결과가 명확** — A condition 에서 ta-02 proxy 61~78 (Δ17), tr-10 proxy 40~54 (Δ14) — 단일
   샘플 noise 가 r3→r4 평균차(-7.8) 보다 큰 자릿수임이 한눈에 보이는 데이터.
4. **즉시 적용 가능한 fix** — temperature/seed 두 라인 추가만으로 결정성 ~6배 개선. ADR 으로 정책
   까지 박제.

## 아쉬운 점

1. **현실의 30-prompt 풀 N=3 재채점은 미수행** — 본 task spec 은 RCA 까지였으나, 진짜 r3→r4 회귀가
   noise 였는지 확정하려면 r4 PNG 셋 (gitignored) 으로 동일 30 호출을 N=3 으로 다시 돌려야 함. 다음
   r5 owner 의 일.
2. **gpt-4o `seed` 의 약속 강도 미검증** — OpenAI docs 상 best-effort 이고 `system_fingerprint` 가
   변하면 결과 drift 가능. 본 회차에서는 단기 (수분 내) 호출이라 stable 했지만, 회차 간(1주 후) drift
   재현 실험은 향후 별도.
3. **카테고리별 sample size 불충분** — n=6/카테고리 에선 카테고리 평균의 SE 가 ~4 점이라 카테고리
   단위 회귀 라벨 자체에 한계. ADR 에 Δavg<15 라벨 금지 로 우회했으나 본질적 fix 는 prompt set 확장.
4. **TM-66 마이그레이션 시점에 결정성 점검을 못 함** — Opus 4.7 → gpt-4o 전환 PR 에서 temperature
   결정성을 고려했으면 TM-46 r3 부터 stable 했을 것. 마이그레이션 PR 에 "결정성 회귀 체크" 항목을
   review 체크리스트로 추가 필요.
