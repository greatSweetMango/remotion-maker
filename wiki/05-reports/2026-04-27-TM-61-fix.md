# TM-61 — fluid-blobs metaball 합성 미발현 수정

## 요약

TM-43 시각 검수에서 발견된 `fluid-blobs` 템플릿의 시작 시점 metaball 합성 누락 버그. 표준 SVG goo 필터 파이프라인의 마지막 단계가 잘못 구성되어 있어, blur 가 약하거나 blob 거리가 멀 때 항상 개별 원으로 보였음. 필터 종단을 `feBlend(SourceGraphic over goo)` → `feComposite(operator="atop")` 로 교체하고 `stdDeviation` 의 하한을 blob 크기에 비례하게 두어 문제를 해결.

## 근본 원인

`src/remotion/templates/FluidBlobs.tsx`의 `<filter id="goo">`:

```diff
- <feBlend in="SourceGraphic" in2="goo" />
+ <feComposite in="SourceGraphic" in2="goo" operator="atop" />
```

원래 코드의 `feBlend` 는 `mode` 미지정 → 기본 `normal` → **un-blurred SourceGraphic 이 threshold 결과(goo) 위에 그대로 그려지는 효과**. 이 때문에 metaball 합성 결과(인접 원이 융합된 부드러운 윤곽)가 항상 원본 원으로 가려짐. blur 가 충분히 크고 blob 들이 가까울 때만 SourceGraphic 의 외곽 너머로 halo 가 새어 나와서 mid 시점에는 합성된 것처럼 보였던 것 (TM-43 audit이 관찰한 정확한 현상).

`feComposite operator="atop"` 으로 바꾸면:
- 베이스: thresholded goo (메타볼 윤곽이 살아남)
- 위에: SourceGraphic 의 색상이지만 goo 알파 영역에만 — 즉 윤곽은 goo 가 결정, 색상은 원본이 결정. 이것이 canonical SVG goo recipe.

추가 안전망:

```ts
const effectiveBlur = Math.max(blur as number, Math.round(size * 0.18));
```

PARAMS 슬라이더에서 사용자가 blur 를 작게 (10px) 두더라도 blob 크기 대비 최소 18% 의 stdDeviation 을 보장 → metaball 효과가 사라지지 않음.
