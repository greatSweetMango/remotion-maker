# TM-107 — Per-sequence panel: counts, header label, helpers

## TL;DR

TM-107의 핵심 분기 로직(시퀀스별 PARAMS 필터링)은 이미 TM-28/ADR-0008에서 출시되어 있었음. 본 세션은 그 위에 (a) 시퀀스별 파라미터 개수 뱃지 (b) CustomizePanel 헤더의 활성 시퀀스 라벨 (c) 두 헬퍼(`countParamsBySequence`, `activeSequenceLabel`) + 단위 테스트를 추가하여 "지금 어떤 시퀀스를 편집 중인가"를 항상 명확히 노출.

## 무엇이 바뀌었나

- `src/lib/sequences.ts`: `countParamsBySequence(parameters, segments)` — 글로벌 파라미터를 각 시퀀스에 합산하여 표시 카운트를 정확히 반영. `activeSequenceLabel(segments, activeId)` — ALL/없음 시 null.
- `src/components/studio/SequenceTimeline.tsx`: `parameters` 옵셔널 prop 추가. 각 시퀀스 탭에 "Label N" 형태로 카운트를 노출하고 `data-sequence-id`/`data-active` 데이터 속성을 부여 (E2E/디버깅).
- `src/components/studio/CustomizePanel.tsx`: 헤더 텍스트가 `Customize · Feature 1` 처럼 활성 시퀀스를 표시 (멀티 시퀀스 한정). 단일 시퀀스/no-context 케이스는 기존과 동일하게 `Customize`만.
- `__tests__/lib/sequences.test.ts`: 새 헬퍼 2개에 대한 13개 신규 케이스. 27/27 그린.

## 왜 / 배경

TM-107 task spec은 "각 시퀀스별 패널 = 해당 시퀀스 PARAMS만" + "Sidebar 와 selectedSequenceId 양방향 동기화"를 요구. 코드 인스펙션 결과 `ActiveSequenceProvider`(TM-28) + `filterParamsForSequence`(ADR-0008)가 이미 본 요구를 만족하고 있음. 따라서 본 작업은 "스펙은 끝났지만 사용자에게 보이지 않는 부분"을 보강하는 데 집중:

1. 사용자가 현재 어떤 시퀀스를 편집 중인지 패널 헤더에서 항상 보이게 (시각적 컨텍스트)
2. 어떤 시퀀스에 편집 가능한 노브가 몇 개인지 timeline 탭에서 즉시 파악 (네비게이션 어포던스)
