# TM-69 — OpenAI `response_format: json_object` 강제

## TL;DR

TM-41 r2 QA에서 ta-09 등 일부 케이스가 OpenAI 응답을 `JSON.parse` 하지 못해 실패했다. system prompt의 "respond with JSON" 지시만으로는 gpt-4o / gpt-4o-mini가 마크다운 코드 펜스나 prose를 섞어 보낼 가능성이 남아 있었다. `src/lib/ai/client.ts` OpenAI 분기에 `response_format: { type: 'json_object' }`를 추가해 모델 차원에서 JSON-only 출력을 강제하고, system prompt에도 "Respond strictly in JSON" 한 줄을 보강했다. 단위 테스트 7건(기존 3 + TM-69 4) 그린, 라이브 5콜 5/5 PASS.

## 무엇이 바뀌었나

- `src/lib/ai/client.ts`
  - OpenAI 분기에 `response_format: { type: 'json_object' }` 추가.
  - system prompt에 `\bjson\b` 키워드가 없으면 `Respond strictly in JSON.`을 자동 보강 (OpenAI json mode 요건).
  - Anthropic 분기는 무변경 — Claude는 system prompt의 JSON 지시를 잘 따르고 별도 JSON mode가 없다.
- `src/lib/ai/prompts.ts`
  - `GENERATION_SYSTEM_PROMPT` 끝에 "Respond strictly in JSON. The entire response is parsed by JSON.parse — any non-JSON characters break the pipeline." 추가.
- `__tests__/lib/ai/client-stream.test.ts`
  - TM-69 describe 블록 추가 (4 tests):
    1. `response_format=json_object`가 OpenAI에 전달됨
    2. system prompt에 "json"이 없을 때 자동 보강
    3. valid JSON 스트림은 round-trip JSON.parse 성공
    4. non-JSON 스트림은 JSON.parse가 throw (호출자가 에러로 surface)
- `__tests__/qa/tm-69-json-live.test.ts` (신규, TM69=1 게이트)
  - 5개 짧은 프롬프트로 gpt-4o-mini 라이브 호출, 모두 valid JSON.

## 왜 / 배경

TM-41 r2 결과에서 ta-09 invalid JSON 실패가 보고됐다 ([[2026-04-27-TM-41-qa|TM-41 QA]]). 원인은 OpenAI 응답이 system prompt 지시를 위반하고 prose 또는 fenced markdown을 섞어 출력한 것. gpt-4o(-mini)는 [json mode](https://platform.openai.com/docs/guides/json-mode)를 지원하므로 가장 확실한 방어는 API 차원에서 제약하는 것이다. json mode는 system/user 메시지 어딘가에 "json"이라는 단어가 있어야 동작하는데, 기존 GENERATION_SYSTEM_PROMPT는 이미 그 조건을 만족했고, custom system prompt를 쓰는 향후 호출에 대비해 클라이언트가 자동으로 키워드를 보강하도록 했다.

## 영향

- 코드 / 시스템: OpenAI 호출만 수정 (Anthropic 무영향). `chatCompleteStream`의 외부 시그니처 무변경.
- 사용자: invalid JSON 실패율 ↓. 5개 라이브 케이스 5/5 PASS, 대부분 8–12s 응답.
- 비용: json mode는 OpenAI에서 추가 비용 없음. 라이브 검증 ~$0.01 미만 사용.
- 회귀: streaming 동작 그대로. response_format은 streaming과 호환됨 (OpenAI 공식 가이드 확인).
