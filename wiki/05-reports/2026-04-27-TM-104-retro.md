# TM-104 retro — r1

**Verdict**: APPROVE — TM-102 pipeline에 비침투적으로 long-form 지원 추가. 30/30 tests, 0 regressions.

## What worked
- TM-102 ADR-0020 구조가 깨끗해서 stage 1(outline)에만 hook을 끼면 됨. stage 2/3는 무수정.
- Duration hint extraction을 정규식 기반 휴리스틱으로 처리 — KO/EN/혼합 모두 한 함수에서. LLM call 추가 없음 (cost 0).
- `enforceScenePlan` 사후 보정 패턴: LLM이 directive를 무시해도 결과가 사용자 의도와 일치 보장. AI 비결정성을 deterministic fence로 감쌈.
- 기존 "caps at 4" 테스트가 4→12 변경을 즉시 캐치해줌 → 회귀 자신감.

## What didn't
- 처음에 LLM에게만 의지하려 했으나 실제 LLM(특히 Sonnet)은 directive를 따르지만 frame 산수에 ±몇십 frame 어긋남이 종종 발생 → fps 1초 tolerance + 강제 재배분으로 가드.
- `MAX_SCENES=12` 결정은 임의적 (실측 미확보). 8씬 = 120s @ 15s/scene가 sweet spot이라고 추정. 후속 telemetry 필요.
- 라이브 검증은 비용/시간 ROI 낮아 skip — multi-step 파이프라인 자체는 TM-102 r7에서 이미 검증됐고 TM-104는 입력/출력 길이만 확장.

## Lessons
- **AI fence 패턴**: prompt directive + post-fix 이중화. directive로 80% 잡고, post-fix로 20% drift 흡수.
- **휴리스틱 우선**: 자연어 duration 추출은 정규식으로 충분. LLM call로 가지 말 것 (cost / latency / 신뢰성 모두 손해).
- **캡 제거 ≠ 무한 확장**: 4 → 12로 올린 건 Promise.all fan-out 한계 + 각 씬 평균 길이 5–15s 가정. 그 이상은 별도 chunked render path가 필요.
