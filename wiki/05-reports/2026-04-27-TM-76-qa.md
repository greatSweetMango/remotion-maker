## 왜 / 배경

- ADR-0010 introduced `sourceAssetId` self-relation on `Asset` for fork lineage (TM-23). Until TM-76 the live behavior had never been exercised end-to-end against a running server with two distinct sessions.
- task spec required: upload→version→fork→authz→delete cascade verification with raw SQL orphan check.

## 영향

- **Code**: no asset/fork/edit/share/delete code changes — behavior matches spec.
- **Infra fix**: `scripts/setup-worktree.sh` now pins `DATABASE_URL` to an absolute path (see tech-note). This unblocks future worktree-based QA runs that previously read from main repo's DB by accident.
- **Schema annotations**: live-verified `onDelete: Cascade` (AssetVersion) and `onDelete: SetNull` (Asset.sourceAsset) — matches schema.
