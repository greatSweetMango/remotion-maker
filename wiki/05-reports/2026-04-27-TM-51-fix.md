# TM-51 — PRO 모델 빈 코드 응답 fix

## TL;DR

- TM-41 QA 에서 발견된 PRO tier (gpt-4o) 4건의 25-char stub (`const Component = () => null;`) 패턴을 차단.
- 대응: post-validate 단(`detectPlaceholderCode`) 추가 + reinforced system prompt 로 1회 재시도 + 두번째도 placeholder 면 user-facing error.
- 신규 의존성 0, 기존 26개 테스트 + 신규 6개 테스트 (총 274 pass / 1 skip).

## 문제

TM-41 QA (`wiki/05-reports/2026-04-27-TM-41-qa.md`):

| ID | tier | category | code_length | 모드 |
|---|---|---|---:|---|
| ld-01 | PRO | loader | 25 | generate |
| ld-02 | PRO | loader | 25 | generate |
| ta-01 | PRO | text-anim | 25 | generate |
| tr-01 | PRO | transition | 25 | generate |

응답 형태: `{"mode":"generate","code":"const Component = () => null;",...}`. sandbox `validateCode` 통과 (deny-list 위반 없음), PARAMS 추출 0개, 화면 빈 상태로 customize 진입.

## 원인 분석

1. **deny-list 만 통과 검증**: `sandbox.validateCode` 는 보안 패턴(eval/fetch/...)만 본다. **structural** 검증(PARAMS 존재, JSX 본문, 본문 길이)은 어디에도 없었음.
2. **prompt 가 stub 을 거부 안 함**: `GENERATION_SYSTEM_PROMPT` 는 PARAMS / JSX 형식을 안내하지만 "빈 stub은 무효" 라는 negative example 없음. gpt-4o 가 빠른 응답(1.0~1.4s)을 위해 형식만 맞춘 stub 을 반환.
3. **재시도 없음**: 한 번의 응답으로 그대로 customize 페이지에 표시.

## 조치

### 1. post-validate 추가 (`src/lib/ai/generate.ts`)

신규 export `detectPlaceholderCode(code: string): string[]` — 다음 4개 휴리스틱:

- 길이 < `PLACEHOLDER_MIN_CODE_LENGTH` (200 chars). 관측된 stub 25자 vs 정상 코드 ~600+자.
- `const PARAMS = ...` 선언 누락 (ADR-0002 위반).
- JSX 요소 부재 — 정규식 `<[A-Z][A-Za-z0-9]*[\s/>]` 또는 `<AbsoluteFill\b`.
- `=> null` 본문 + 전체 길이 < 400 (canonical stub 전용).

reasons 배열을 반환 — 빈 배열 = 통과. 어떤 항목이 걸렸는지 로그/에러 메시지에 활용.

### 2. system prompt 보강 (`src/lib/ai/prompts.ts`)

신규 export `GENERATION_NON_EMPTY_REINFORCEMENT` — 재시도 시에만 prepend.

핵심 문구:
- `const PARAMS = {} as const` alone is NOT acceptable
- A bare `return null` body is FORBIDDEN
- Code MUST be ≥ 200 chars
- "If too vague, switch to mode=clarify instead of stub"

첫 호출은 기존 `GENERATION_WITH_CLARIFY_SYSTEM_PROMPT` 그대로 — token 사용 증가는 retry 시점에만.

### 3. retry-once 로직

`generateAsset` 를 helper `generateOnce` 로 분리:
- 1차 호출: 기본 system prompt
- response → 그대로 반환
- placeholder → reasons 와 함께 caller 에 반환
- 2차 호출 (placeholder 시): `GENERATION_WITH_CLARIFY_SYSTEM_PROMPT + GENERATION_NON_EMPTY_REINFORCEMENT`
- 다시 placeholder → user-facing Error throw: `"AI returned a placeholder/empty component twice. Please rephrase…"`

dev 환경에선 첫 placeholder 시 `console.warn` 으로 reasons 출력 — 운영시 silent.

### 4. 테스트 (`__tests__/lib/ai/generate.test.ts`)

기존 stub code 가 우리 새 validator 에 걸리므로 substantive code 로 교체. 신규 추가:

- `detectPlaceholderCode` unit (4 케이스): canonical stub 거부, empty PARAMS 거부, no-JSX 거부, substantive 통과
- `generateAsset retry on placeholder` (2 케이스): retry 후 성공, retry 후 또 stub → throw

총 신규 6 tests, 회귀 0.
