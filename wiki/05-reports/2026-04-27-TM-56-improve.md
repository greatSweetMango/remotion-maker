# TM-56 — 워크트리 env 자동 부트스트랩

## TL;DR

병렬 워크트리 생성 시 매번 `.env.local` 수동 복사 + `NEXTAUTH_URL` 포트 치환 + `prisma db push` 반복하던 패턴을 `scripts/setup-worktree.sh`로 단일 명령화. `.claude/commands/orchestrate.md` Step 3에서 `git worktree add` 직후 자동 호출되도록 통합. idempotent 검증 통과.

## 무엇이 바뀌었나

- **새 스크립트**: `scripts/setup-worktree.sh <worktree_path> <dev_port>` — 메인 repo의 `.env.local`을 워크트리로 복사 → `NEXTAUTH_URL`을 `http://localhost:<dev_port>`로 치환 → 워크트리에서 `npx prisma db push --skip-generate` 실행 (각 워크트리가 자체 SQLite `dev.db` 보유) → 완료 배너.
- **Orchestrator 통합**: `.claude/commands/orchestrate.md` Step 3에서 `git worktree add` 직후 본 스크립트 호출. `dev_port` 컨벤션은 `30NN` (NN = task_id 두자리, 예: TM-56 → 3056).
- **TeamLead 프롬프트 갱신**: `prompts/team-lead.md` 전제 문단에 "Orchestrator가 이미 setup-worktree.sh 실행 완료" 명시. 누락 시 idempotent 재실행 가이드.

## 왜 / 배경

TM-42/TM-43/TM-45 retro에서 동일한 문제가 반복 보고됨:
- TeamLead가 워크트리에서 `npm run dev` 시 `NEXTAUTH_URL`이 main의 3000 그대로여서 OAuth 콜백 충돌
- `prisma/dev.db`가 없어 first-request 시 500 → 매번 수동 `prisma db push`
- 디버깅에 5~10분씩 소모, 야간 자동 루프에서 silent 실패

수동 작업을 단일 스크립트로 흡수하여 Orchestrator가 워크트리 생성과 동시에 호출 → TeamLead 진입 시점에 이미 dev-ready.

## 영향

- **개발 흐름**: 워크트리 생성 → 즉시 `PORT=30NN npm run dev` 가능. 평균 setup 시간 5~10분 → 0초 (자동).
- **에이전트 컴퍼니**: 야간 무인 루프 안정성 향상. TM-42/43/45 류 회고에서 반복되던 "env 누락" 클래스 이슈 제거.
- **신규 의존성 0**: 순수 bash + 기존 `npx prisma`. 추가 npm 패키지 없음.
- **idempotent**: 같은 워크트리에서 재실행해도 안전 (`.env.local` 덮어쓰기, `db push`는 schema 동기화).

## 검증

- 본 워크트리(`worktrees/TM-56-worktree-env-bootstrap`)에서 스크립트 1차 실행 → `.env.local` 생성, `NEXTAUTH_URL=http://localhost:3056` 치환 확인, `prisma/dev.db` 생성.
- 2차 실행 → "database is already in sync" / NEXTAUTH_URL 동일 유지 → idempotent 확인.
