# TM-50 — CSP `media-src data:` fix

## Context

TM-41 (AI 생성 E2E QA) 와 TM-43 (template visual audit) 가 customize 페이지 콘솔에서 동일 CSP violation 을 175회+ 관측했다.

```
Refused to load media from 'data:audio/wav;base64,...' because it violates the following Content Security Policy directive: "media-src 'self' blob: https:".
```

원인은 Remotion preview 가 audio 트랙 probe 시 base64-encoded silent WAV stub 을 `<audio src="data:audio/wav;base64,...">` 로 마운트한다는 점. `next.config.ts` 의 `SANDBOX_CSP` 가 `media-src` 에 `data:` 를 포함하지 않아 모든 probe 가 차단됐다.

기능 자체는 동작했지만 (probe 결과 무시 → 무음 fallback), 콘솔 노이즈 175회/페이지로 실제 audio 오류를 묻고 QA 트리아지를 어렵게 만들었다.

## Change

`next.config.ts` 의 단일 라인 변경:

```diff
- "media-src 'self' blob: https:",
+ "media-src 'self' data: blob: https:",
```

추가 보안 영향: `data:` URI 는 outbound 네트워크 호출이 아니므로 데이터 유출 벡터가 아니다. 동일 정책을 `img-src` (`data: blob: https:`) 와 `font-src` (`data:`) 에서 이미 허용하고 있어 일관성이 오히려 회복된다. ADR-PENDING-TM-34 의 sandbox hardening 의도(`connect-src` 좁힘 + `frame-ancestors 'none'`)는 그대로 유지된다.
