# TM-65 retro

## What

Made `scripts/setup-worktree.sh` host-aware: defaults to `127.0.0.1` and accepts an
optional explicit hostname, fixing the cross-worktree cookie clobber surfaced in TM-43.

## What went well

- Single-file change. No app code, no NextAuth config edits, no migrations.
- Backward-compatible: existing `localhost:*` `.env.local` files keep working;
  re-running the script promotes them to the new convention.
- Hostname validation guard catches typos at bootstrap time, not at runtime.

## What was tricky

- Tempting to "fix it in NextAuth" by suffixing cookie names per port. Resisting that
  was the right call — it would have leaked dev-only complexity into prod auth code.

## Follow-ups (not in scope here)

- Document the hostname convention in `wiki/02-dev/architecture.md` once we add a
  "running multiple worktrees" section. Tracked as a docs nit, not a task.
- If `*.local` aliases prove popular, consider auto-suggesting `tm-<id>.local`
  inside `setup-worktree.sh` based on the worktree directory name — keep that
  optional to avoid /etc/hosts surprises.

Released (TM-65 is independent infra; no triggers_requalify).
