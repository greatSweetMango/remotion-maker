# TM-42 — AI 편집 플로우 E2E QA (1회차, 부분 완료)

## TL;DR

- 20 세트 중 **4 세트 완료** (12 edit turn) 후 인터럽트로 조기 종료. 남은 16 세트는 `AI-FOLLOWUP-TM-42-remaining-sets` 로 r2 위임.
- 측정된 4 세트 범위 안에서 **Acceptance 조건 만족**:
  - PARAMS 손실률 0/12 (완벽 보존)
  - 의도 외 변경 2/12 turns (16.7%) — 모두 `scene` (장면 추가) turn 에서 부수 키 변경 → 사양상 허용 범위
- `cache_control` 히트율은 **N/A** — 환경이 `AI_PROVIDER=openai` 라 ADR-0003 의 Anthropic ephemeral cache 비활성. 대체로 cold/warm latency 측정.

### Latency (ms)

| Turn | n | avg | p50 | min | max |
|---|---|---|---|---|---|
| generate (cold) | 4 | 5,397 | 6,250 | 4,121 | 6,290 |
| edit_color (warm-1) | 4 | 4,726 | 4,901 | 3,724 | 5,477 |
| edit_scene (warm-2) | 4 | 7,197 | 7,319 | 5,658 | 9,590 |
| edit_speed (warm-3) | 4 | 7,250 | 7,398 | 5,834 | 8,523 |

**Latency 해석** (cache 대체 지표):
- `edit_color` 가 `generate` 대비 약 **12.4% 단축** (5397→4726ms). 동일 어시스턴트 prefix 재사용에 의한 OpenAI 백엔드 자체 캐시(implicit) 효과로 추정.
- `edit_scene` / `edit_speed` 는 오히려 길어짐 — 코드 길이 누적 (장면 추가로 토큰 증가) 영향.
- → OpenAI Chat Completions API 는 명시적 cache_control 미지원이라 ADR-0003 효과 측정 불가. **Anthropic provider 모드에서 재측정 필요** (별도 task).

### PARAMS 보존 (deep-equal)

| Turn | 의도 키 | 손실 | 의도 외 변경 |
|---|---|---|---|
| color | `primaryColor` | 0/4 | 0/4 |
| scene | `__code_growth__` (n/a) | 0/4 | 2/4 |

`scene` turn 에서 발생한 2 건의 "의도 외 변경" 은 LLM 이 새 scene 추가에 맞춰 기존 `text` / `fontSize` 같은 키 값을 미세 조정한 케이스 (예: `"Counter"` → `"Counter Animation"`). 사양상 scene 추가는 PARAMS 부수 변경 허용. **버그 아님**.

## 발견 사항 (코드 분석)

### 1. ADR-0003 (Prompt Caching) 가 OpenAI provider 시 사실상 무효
- `src/lib/ai/client.ts:50-66` OpenAI 분기에서 `AIContentPart.cache` 플래그를 무시하고 `text` 만 join 하여 string 으로 전송.
- `src/lib/ai/prompts.ts:189` `buildEditMessages` 가 EXISTING CODE 블록에 `cache: true` 마킹하지만 OpenAI 모드에선 그냥 평문.
- **권고**: ADR-0003 본문에 "Anthropic only" 명시 추가 또는 OpenAI Prompt Caching (자동, 1024+ tokens 정적 prefix) 활용 가이드 추가.

### 2. Anthropic provider 코드 경로에서도 system prompt 에 cache_control 미적용
- `client.ts:88` `client.messages.create({ system, ... })` 가 system 을 string 으로만 전달.
- ADR-0003 은 "system prompt + prior code" 양쪽 캐싱 명시 → 현재 system prompt 캐싱 누락. (Anthropic provider 활성 시 토큰 비용 영향)
- 본 task 범위 외 → fix task 후보.

### 3. dev 환경 설정 누락
- 워크트리에 `.env.local` 미복사 → 수동 복사 필요.
- `NEXTAUTH_URL` 이 3000 hardcoded → 3042 포트로 수정 필요.
- **권고**: `worktree-bootstrap` 스크립트에 env 복사 + port 자동 치환 추가 (별도 infra task).
