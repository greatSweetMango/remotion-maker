# TM-103 — URL 컨텍스트 인제스트 (feature)

## What shipped

레퍼런스 URL을 첨부하면 OG meta + h1/h2 + 색상 토큰을 추출해서 generate prompt 에 자동 주입하는 인제스트 파이프라인.

- `src/lib/ingest/url.ts` — cheerio 기반 파서 (server-only). SSRF 가드 + 6s timeout + 1.5 MB body cap
- `src/lib/ingest/format.ts` — client/server 공용 포매터 (의존성 없음, prompt-cache 안전)
- `src/app/api/ingest/url/route.ts` — `POST /api/ingest/url`, auth 필수, quota 비차감
- `src/components/studio/PromptPanel.tsx` — "Attach a reference URL" 토글 + 입력 + 미리보기 칩 (썸네일/타이틀/팔레트/제거 X)
- `src/hooks/useStudio.ts` — `attachedContext` state + `attachUrl()` + generate 시 user message 끝에 `[ATTACHED CONTEXT]` 블록 append
- `__tests__/ingest/url.test.ts` — 14 tests (URL 검증, OG/headline 파싱, 팔레트 추출, formatter)

## Decisions

- ADR-PENDING-TM-103 — server route + client formatter 분리 이유 (cheerio ESM 충돌), system prompt 가 아닌 user message 에 append 하는 이유 (ADR-0003 cache key 보존)
- 이미지 multimodal vibe 추출은 v2 로 미룸 (cost·latency 검증 후)

## Gotchas (recorded)

- `cheerio` v1 의 `dist/browser/index.js` 가 raw `export` 구문을 가지고 있어서, 동일 모듈 그래프에 client 코드 (jest 의 jsdom 환경 포함) 가 transitive 로 끼어들면 `SyntaxError: Unexpected token 'export'` 로 터짐.
  - 해결: 포매터를 `format.ts` 로 분리 → `useStudio` (client) 는 format 만 import, server route 는 url.ts 전체 import.
  - 발견 경로: `tm-78-r2-undo-redo-live.test.tsx` 회귀 (1 suite 추가 실패 → diff 로 격리 → 분리 후 baseline 동일).
