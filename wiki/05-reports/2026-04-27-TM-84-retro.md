# TM-84 Retrospective — mobile/tablet customize QA

## TL;DR

15분 timeout 안에 5/35 templates × 2 viewport 검증 완료. 100% 일관 결함 발견 → 전수 검증 없이 root layout 가설 확정. Fix task TM-80 spawn 으로 escalation.

## 잘된 점 (Keep)

- 첫 templates 1-2 만으로 패턴 100% 일관 확인 → 35 전수 대신 5 sample 로 시간 절약 합리화.
- `browser_evaluate` 로 `div[class*=remotion]` 의 BoundingClientRect 직접 측정 → 시각만으로 놓칠 0×0 실측 데이터 확보.
- worktree DB symlink + DATABASE_URL 절대경로 fallback 으로 setup-worktree.sh 의 path 모호성 워크어라운드.

## 아쉬운 점 (Problem)

- `setup-worktree.sh` 가 prisma schema dir 기준(`prisma/dev.db`)으로 push 하는데 Next 런타임은 cwd 기준(`./dev.db`)으로 해석 → User table not found 에러로 30s 손실. 다른 worktree 도 같은 함정 가능.
- `task-master update-task --append` 가 PERPLEXITY_API_KEY 미설정 시 실패. metadata 박제는 결국 `tasks.json` 직접 수정으로 우회.
- Playwright MCP `browser_resize` 가 `browser_navigate` 후 reset 되는 듯 — 매 nav 마다 resize 재호출 필요 (workflow overhead).
- iPhone 캡처가 `fullPage:true` 와 `false` 모두 동일 보임 → mobile 의 진짜 layout (스크롤 가능 영역) 일부 못 봤을 가능성. r2 에서는 viewport-only + scroll 이미지 분리 필요.

## 시도해볼 것 (Try)

1. `setup-worktree.sh` 가 `.env.local` 에 **절대경로 DATABASE_URL** 박도록 패치 (별도 task — TM-84 retro 에서만 노트). Phase D 에서 같이 PR 에 안 넣음 (scope creep).
2. r2 회차에서 35 전수 검증을 위해 작은 Playwright 스크립트(`scripts/qa-mobile-templates.ts`) 작성 — 실측 결과 JSON 으로 떨굼.
