# TM-90 — Customize panel a11y fixes

## Context

TM-80's axe-core audit reported **critical=21 / serious=77** violations
across the studio. The dominant categories were all in the customize panel:

1. `<input type="number">` inside `ParameterControl` had no accessible name.
2. shadcn `<Switch>` rendered without an `aria-label`/`aria-labelledby`.
3. radix `<Slider>` thumb (the actual focusable element) had no accessible
   name — radix does not auto-forward labels from the Root.
4. `text-slate-500` (#64748b) on `bg-slate-900` (#0f172a) → contrast ratio
   ~4.18, fails WCAG AA for normal text.

## Changes

### `src/components/studio/ParameterControl.tsx`
- Added `useId()`-based `labelId`. Every visible `<Label>` now carries that
  id and the matching control receives `aria-labelledby={labelId}` (slider
  thumb, switch, text input, select trigger, color popover trigger, icon
  picker trigger, upload-asset select).
- Number input (range param) gets `aria-label="<label> value (<unit>)"` —
  it lives next to the slider, so a separate accessible name is more useful
  than reusing the slider's label.
- Hex color input inside the color popover gets `aria-label="<label> hex
  color"`.
- Icon-search input gets `aria-label="Search icons"`.
- All `text-slate-500` text content inside this file bumped to
  `text-slate-400` for AA contrast on the dark panel background. Decorative
  icons marked `aria-hidden`.

### `src/components/ui/slider.tsx`
- Slider now accepts `aria-label` / `aria-labelledby` and forwards them to
  every `<SliderPrimitive.Thumb>`. Required because radix Slider does not
  propagate Root labels to the thumb (which is the focused element).

### `src/components/studio/CustomizePanel.tsx`
- Group headers (`Resources`, `Colors`, `Size`, …) bumped from
  `text-slate-500` → `text-slate-400`. Decorative group icons marked
  `aria-hidden`.
- "No parameters defined…" hint and details `<summary>` bumped to
  `text-slate-400` / `hover:text-slate-200`.
