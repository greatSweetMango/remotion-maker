# TM-85 Fix — Sandbox fuzz hardening

## 무엇을 했나

`src/lib/remotion/sandbox.ts` deny list에 8개 카테고리 신규 패턴 추가 +
0-day DoS 2종 (setTimeout 자기재귀, Promise 무한 chain) 정적 차단:

- **Reflect / Proxy**: `Reflect.get/construct`, `new Proxy`, `Proxy(...)`, `Proxy.revocable` 모두 차단.
- **Encoding helpers**: `atob`, `btoa`, `Buffer` — `eval(atob('...'))` 우회 차단.
- **Realm escape**: 기존 `globalThis` + `process` 위에 `global` 토큰 word-boundary 추가.
- **WebAssembly**: 별도 실행 surface — 토큰 단위 통째 deny.
- **Worker family**: 기존 `Worker/SharedWorker` + `ServiceWorker` 보강.
- **Timers**: `setTimeout`/`setInterval`/`requestAnimationFrame`/`queueMicrotask` 호출 자체를
  전부 deny. Remotion 컴포넌트는 `useCurrentFrame` 기반이므로 사용자 코드에서
  타이머가 등장하면 100% recursive-DoS 또는 microtask 폭주 신호로 간주.
- **Recursive Promise chain (0-day)**: 정규식 한 줄로는 안 되어
  `detectRecursivePromiseChain()` 휴리스틱 함수 추가 — 모든 함수/화살표 선언명을
  수집한 뒤 `.then(<own-name>)` 패턴이 등장하면 차단. `Promise` 토큰 co-location도
  요구해서 일반 named-callback `.then(handler)`은 통과.

신규 fuzz 케이스 34개 (`__tests__/lib/evaluator/sandbox-fuzz.test.ts`) — 30+ 차단
케이스 + 4개 benign baseline (recursive 아닌 `Promise.then`, `useGlobals`처럼
`global` 부분문자열 포함 식별자 등 false-positive 회귀 가드).

## 결과

- jest: **82 / 82 sandbox+evaluator passed** (3 suites). 전체 `npx jest`: 427 passed,
  3 skipped, 0 failed.
- TypeScript: 신규 에러 0 (기존 evaluator-fuzz.test.ts의 BigInt/regex flag
  pre-existing 에러는 손대지 않음).
- OpenAI 호출: **0**.

## 설계 노트 — 왜 AST 풀파서가 아닌가

TM-48 회고에서 언급된 "AST 기반으로 교체" 후보를 검토했으나, 이번 라운드 범위는
"deny list에 카테고리 신규 추가"라 sucrase output 위에 별도 traverse를 끼우면
편집-경로 1tick 비용이 늘어남. 정적 regex + 1개 휴리스틱 함수로도 fuzz 케이스
30+ 모두 잡히고 false-positive 베이스라인 4종 통과. AST 전환은 별도 task로 유지.

## 사용자 향 메시지 검증

`validateCode`는 항상 `errors[]`에 `Forbidden: <label>` 형식 단문을 넣음. 호출처
(`PlayerPanel`, `TemplatePicker` 등)는 이 라벨을 그대로 토스트에 노출해도
스택 트레이스 누수 없음. fuzz 테스트가 길이(<120) + stack-frame 패턴 부재를
케이스마다 assert.

## 0-day 분석

1. **setTimeout 자기재귀** — `function loop(){ setTimeout(loop, 0) }` 는
   evaluator의 `Date.now()` 사후 측정이 실행을 중단시키지 못하므로 사전 차단
   필수. timer 자체를 deny해서 우회 surface 봉쇄.
2. **Promise.then 무한 chain** — `Promise.resolve().then(f).then(f)…` microtask
   큐를 영원히 채워 탭 freeze. `Promise` 토큰만 deny하면 정당 코드(데이터 fetch
   체인)도 막혀서 over-block. 자기재귀 패턴만 골라내는 휴리스틱이 적정 균형.

## Follow-ups

- AST 기반 deny 평가 (별도 research task) — 현재 휴리스틱은 완벽치 않음
  (mutual recursion `f→g→f` 미감지). 단, fuzz/실 LLM output에서 그 변종은 미관측.
- evaluator의 사용자 메시지를 sandbox `errors[]` 라벨과 sync — 현재는 sandbox 단
  pre-flight에서 차단되면 호출처가 자체 토스트를 띄움. 통일된 에러 채널 후보.
