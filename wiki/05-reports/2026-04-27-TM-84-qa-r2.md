# TM-84 r2 — Studio (customize) 페이지 mobile/tablet 재검증 (TM-98 fix 후)

## TL;DR

TM-98 (PR #96) 머지로 r1에서 보고된 3가지 P1 결함이 모두 해소됨을 5 templates × 3 viewport (375 / 768 / 1280) 매트릭스로 재검증. **F-1, F-2, F-3 = RESOLVED**. 나머지 30 templates 전수 검증은 r3 회차 권장 — 단, 셸 레이아웃 결함이지 template 별 결함이 아니었다는 r1 분석 그대로라면 동일하게 통과할 가능성 높음. ParticlePhysics 의 hydration mismatch 는 layout 무관한 별개 이슈 (Math.random in render).

## 무엇이 바뀌었나

- TM-98 fix 가 머지 (`commit 35494e3`, files: `Studio.tsx`, `PlayerPanel.tsx`, `PromptPanel.tsx`, `tier-limits.ts`)
- r2 회차에서 5 templates × 3 viewport = 15 캡처 수집 (`screenshots/TM-84-r2/{iphone,ipad,desktop}-*.png`)

## 검증 매트릭스

5 templates: counter-animation, bar-chart, particle-physics, holographic-card, glitch-effect (r1 과 동일 셋)

| Template | iPhone 375×812 | iPad 768×1024 | Desktop 1280×800 |
|---|---|---|---|
| counter-animation | Player 343×193, docW=375, no overflow, Customize 탭 반응 OK | Player 736×414, docW=768 | 3-pane OK |
| bar-chart | Player 343×193, no overflow | Player 736×414, no overflow | (sanity skip) |
| particle-physics | Player 343×193, no overflow ⚠ hydration warn | Player 736×414, no overflow ⚠ | (sanity skip) |
| holographic-card | Player 343×193, no overflow | Player 736×414, no overflow | (sanity skip) |

⚠ = ParticlePhysics 의 SSR hydration mismatch (template 내부 `Math.random()` 추정) — layout 무관, 별개 task 권장.

## r1 결함 → r2 상태

### F-1 [P1] iPhone 375 — Remotion Player 0×0 → **RESOLVED**

- TM-98: PlayerPanel 을 모바일 스택 최상단에 **persistent mount** (`hidden flex-col lg:hidden` 분기)
- 측정값: 모든 5 templates 에서 visible Player 343×193 (`getBoundingClientRect.width>0`)
- DOM 상에는 desktop 변형 1개 (`__remotion-player` 0×0, `lg:` 분기로 hidden) + 모바일 변형 1개 (visible) 가 공존하나 hidden 측은 layout 미차지

### F-1 보조 [P1] iPhone 375 — Customize 탭 비반응 → **RESOLVED**

- 모바일 bottom-nav `setMobileTab('customize')` 클릭 → 패널 즉시 전환 확인 (counter-animation 에서 8 params 표시, THEME/COLORS/SIZE/TIMING 그룹 정상)
- 캡처: `iphone-counter-animation-customize.png`

### F-2 [P1] iPad 768 — 우측 Customize 패널 잘림 → **RESOLVED**

- TM-98: 데스크톱 3-pane 분기를 `md:` (>=768) → `lg:` (>=1024) 로 변경. iPad portrait (768) 은 모바일/태블릿 stack 으로 폴백.
- 측정값: docW=768=vw, overflowX=false (5/5 templates)
- iPad 캡처에서 toolbar (`fps · duration · 게이지 · 속도 · 테마 픽커`) 모두 가시

### F-3 [P2] iPad toolbar 컴팩트화 → **RESOLVED**

- TM-98: header + PlayerPanel toolbar 에 `flex-wrap`, `min-w-0`, 보조 chrome 의 responsive hide 적용 — 768 에서 줄바꿈 없이 가로 fit 됨

## 발견 (신규)

### N-1 [P3] ParticlePhysics SSR hydration mismatch (layout 무관)

- 콘솔에 React hydration warning 2건. 서버 렌더된 particle 좌표/opacity 와 클라이언트 값 불일치.
- 원인 가설: 템플릿 내부 `Math.random()` 또는 `Date.now()` 가 render 함수 안에서 호출됨.
- 영향: 시각적 깨짐 없음 (Player iframe 내부 패치 안 됨 경고만), 하지만 향후 SSR seed 화 또는 `'use client'` 강제 검토 필요.
- 별개 task 권장. **TM-84 의 통과 조건과 무관**.

## 영향

- 사용자: 모바일/태블릿 사용자 customize 단계 정상 동작 — PRD "any device" 약속 회복.
- 비즈: ADR-0002 자동 바인딩 UX 가 모든 viewport 에서 작동.
- 후속: 30 templates 전수 검증은 별도 batch r3 (가능하면 자동화 — Playwright headless loop) 으로.
