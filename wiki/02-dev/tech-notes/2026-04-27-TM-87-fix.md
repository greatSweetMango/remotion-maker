# TM-87 — worktree 가 main 의 prisma client 를 사용하던 문제

## 한 줄 요약

`scripts/setup-worktree.sh` 에 `npx prisma generate` 단계를 추가하여 각 worktree
가 자체 generated Prisma client (`node_modules/.prisma/client`) 를 갖게 했다.
TM-46 r6 회고에서 발견된 **worktree dev server 가 main 의 `prisma/dev.db` 를
열어 monthly OpenAI usage limit 을 main user 에게 누적시키던** 문제를 해소.

## 배경 — TM-46 r6 회고에서 표면화

TM-46 r6 (visual judge AI QA 6회차) 실행 중 OpenAI 200 generate/month 한도가
**main user 계정에서** hit. 워크트리에서 dev 서버를 띄우고 평가용 generate 를
돌렸는데도 main 의 dev.db 가 카운트를 누적했다. 임시로 PRO 한도를 100000 으로
상향(`src/lib/usage.ts`) 해 우회했고, 회고에서 별도 infra task (TM-87) 가
필요하다고 적시.

## 근본 원인

1. 기존 `setup-worktree.sh` 는 `prisma db push --skip-generate` 만 실행. 즉
   각 worktree 의 SQLite 파일(`<worktree>/prisma/dev.db`) 은 만들지만 generated
   client 는 만들지 않는다. "node_modules 는 worktree 가 main 과 공유한다" 라는
   가정이었다.
2. 실제로는 worktree 의 `node_modules` 가 비어 있거나(주: 본 워크트리 생성 시점
   에 비어 있는 상태로 부트스트랩됨) `@prisma/client` 패키지가 없는 경우, Node
   module resolution 이 main repo 의 `node_modules` 까지 walk 해서 main 의
   generated client 를 로드한다.
3. 더 심각한 부수효과: worktree 에서 `prisma generate` 를 그냥 실행하면 prisma
   CLI 도 동일한 walk 로직으로 main 의 `@prisma/client` 패키지를 찾아 client 를
   **main 의** `node_modules/.prisma/client` 에 emit. 즉 worktree generate 가
   main 의 client 를 덮어쓴다.
4. 결과적으로 worktree dev 서버는 main 의 generated client + main 의 schema-
   embedded `dev.db` resolution 으로 동작 → usage 카운터가 main user 에 박힘.

## 조치

### `scripts/setup-worktree.sh` Step 4 신설

`prisma db push` 직후 다음을 실행:

1. main 의 `node_modules/@prisma/` 와 `node_modules/.prisma/` 디렉토리를 worktree
   `node_modules/` 로 `cp -R` (기존 사본은 `rm -rf` 로 제거 후 재복사 — 멱등).
2. `pushd <worktree> && DATABASE_URL=file:./dev.db npx --yes prisma generate
   --schema <worktree>/prisma/schema.prisma`.
3. 이 시점에 worktree 의 `node_modules/@prisma/client` 가 이미 존재하므로
   `prisma generate` 의 walk 가 worktree 안에서 멈추고 client 를 worktree 의
   `node_modules/.prisma/client` 에 emit.

### 멱등성

- `cp -R` 전에 `rm -rf` 로 정리 → 재실행 시 동일 결과.
- `prisma generate` 자체가 overwrite-safe.
- 본 워크트리(TM-87) 에서 2회 연속 실행 → 둘 다 `Generated Prisma Client (v6.19.3)
  to ./node_modules/@prisma/client` 메시지로 종료. 산출물 동일.

### 디스크 비용

worktree 당 `node_modules/@prisma` ≈115M + `.prisma` ≈30M = **약 145M 추가**.
2-4 worktree 동시 운영 시 0.3~0.6G 수준. 격리성과 main user usage 보호 가치 대비
허용 가능.

## 검증

- TM-87 worktree 에서 `bash scripts/setup-worktree.sh <path> 3087` 실행 → DONE
  배너까지 정상 종료 + `node_modules/.prisma/client/client.js` 등 emit 확인.
- 동일 명령 재실행 → identical output, 에러 없음 (idempotent).
- main repo 의 `node_modules/.prisma/client/` 에 영향 없음 (worktree 내부에서만
  generate 가 발생하므로 main 의 client 가 덮어쓰이지 않는다는 위 설계의 핵심).
